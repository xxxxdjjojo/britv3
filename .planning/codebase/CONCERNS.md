- Issue: The entire codebase is a default `create-next-app` scaffold with zero application logic. The `docs/` directory contains 16,500+ lines of PRD and epic specifications from a previous v2.0 build (described as "92% production ready" with 319 pages, 649 API endpoints, 266 database tables), but the `src/` directory contains only the two default generated files.
- Files: `src/app/page.tsx` (65 lines - default Next.js template), `src/app/layout.tsx` (34 lines - default layout)
- Impact: Every feature described in the PRD must be built from scratch. There is no reusable code from v2.0 in this repository.
- Fix approach: Prioritize foundational infrastructure (auth, database, routing) before feature development. Follow the epic ordering from the PRD.

**Default Metadata Not Updated:**
- Issue: The app still uses default `create-next-app` metadata values instead of Britestate branding.

- Impact: SEO and branding are incorrect if deployed.
- Fix approach: Update metadata to Britestate branding as part of initial setup.

**Default Page Content:**

- Files: `src/app/page.tsx`
- Impact: No application functionality exists.

- Risk: The PRD specifies multi-role authentication (email/password, Google OAuth, magic links, MFA/2FA) but none is implemented.
- Files: No auth files exist. PRD reference: `docs/brit estate prd 2026.txt` (lines 244-276)
- Current mitigation: None - no protected routes or data exist yet.
- Recommendations: Implement Supabase Auth (as specified in `docs/project memory 2026.txt` ADR-002) before any user-facing features. Set up Row-Level Security policies from the start.

**No Environment Configuration:**
- Risk: No `.env` file or environment variable management exists. The project will require secrets for Supabase, Stripe Connect, Anthropic Claude API, MapTiler, and other integrations referenced in the PRD.
- Files: No `.env` or `.env.example` present
- Current mitigation: None needed yet - no integrations exist.
- Recommendations: Create `.env.example` with all required variable names (no values) as the first integration step. Verify `.env` is in `.gitignore` (it is listed in the default Next.js `.gitignore`).

**No Content Security Policy or Security Headers:**
- Risk: The `next.config.ts` is empty with no security headers configured.
- Files: `next.config.ts` (empty config object)
- Current mitigation: None.
- Recommendations: Add security headers (CSP, HSTS, X-Frame-Options, etc.) in `next.config.ts` before deployment. The PRD mentions GDPR/Legal compliance as Epic 11.

**Future Risk - Bundle Size:**
- Problem: The PRD specifies heavy client-side features (MapLibre maps, AI chat, real-time messaging, complex dashboards for 6+ user roles).
