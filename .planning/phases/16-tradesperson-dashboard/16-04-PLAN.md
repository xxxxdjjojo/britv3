- stripe.accounts.create({ type: 'express', country: 'GB', email: user.email, capabilities: { card_payments: { requested: true }, transfers: { requested: true } } })

  <behavior>
    provider-payment-service.ts:
    - getStripeConnectAccount(supabase, providerId): Fetch stripe_connect_accounts row for provider; return null if not yet created
    - initiateStripeConnect(supabase, providerId, userEmail): If no account exists, calls Stripe API to create Express account, inserts stripe_connect_accounts row, returns stripe_account_id
    - getOnboardingLink(stripeAccountId, origin): Calls stripe.accountLinks.create(), returns {url} for redirect
    - getStripeBalance(stripeAccountId): Calls stripe.balance.retrieve(); returns { available_pence, pending_pence }. Cache result in Redis (60s TTL) per anti-pattern guidance.
    - getPayoutHistory(stripeAccountId, limit=20): Calls stripe.payouts.list(); returns typed payout objects.
    - getTransactionDetail(stripeAccountId, chargeId): Calls stripe.charges.retrieve(); returns transaction with job reference from metadata.

    API routes (server-only, no client-side Stripe calls):
    - POST /api/stripe/connect/create-account: Auth guard → calls initiateStripeConnect → calls getOnboardingLink → returns { onboarding_url }
    - POST /api/stripe/connect/onboarding-link: Auth guard → fetches existing account → calls getOnboardingLink → returns { url }
    - POST /api/stripe/webhooks/connect: Verifies Stripe webhook signature (stripe.webhooks.constructEvent), handles account.updated event → updates stripe_connect_accounts SET onboarding_complete, charges_enabled, payouts_enabled

    CRITICAL: stripe SDK is server-only. Import as: `import Stripe from "stripe"; const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);`
    NEVER import stripe in "use client" components.
  </behavior>
