    **InvoicePreview.tsx** ("use client" — PDF preview):
    Uses @react-pdf/renderer with dynamic import (ssr: false):
    ```typescript
    const PDFViewer = dynamic(
      () => import("@react-pdf/renderer").then(m => ({ default: m.PDFViewer })),

    );
    ```

    Invoice PDF Document (InvoiceDocument component using @react-pdf/renderer):
    - Britestate brand header: forest green (#1B4D3E) bar at top, "BRITESTATE" text
    - "INVOICE" heading, INV number, issue date, due date
    - Provider details: business name, address, VAT number (if set)
    - Client details: name, address
    - Line items table: description, qty, unit price, VAT%, total columns
    - Totals: subtotal, VAT, total in bold
    - Payment terms section
    - Footer: "Generated by Britestate.co.uk" in small text
    - Font: embed Inter (from Google Fonts URL or base64) — do NOT use system fonts

    **API route invoices/route.ts** (POST):
    - Auth guard, zod validation, call generateInvoice service, return 201 with invoice

    **API route invoices/[id]/pdf/route.ts** (GET):
    - Auth guard, fetch invoice with line items
    - Import renderToBuffer from @react-pdf/renderer (server-side rendering — no dynamic import needed here)
    - Generate PDF buffer: `const buffer = await renderToBuffer(<InvoiceDocument invoice={invoice} provider={profile} />)`
    - Return: `new Response(buffer, { headers: { 'Content-Type': 'application/pdf', 'Content-Disposition': 'attachment; filename="invoice-{number}.pdf"' } })`

    CRITICAL: @react-pdf/renderer in API routes does NOT need dynamic import (no browser globals on server). Dynamic import is only needed for <PDFViewer> in client components.
