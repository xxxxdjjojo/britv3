    - getVapidPublicKey() returns NEXT_PUBLIC_VAPID_PUBLIC_KEY env var
    - sendPushNotification with valid subscription calls webpush.sendNotification with JSON payload containing title, body, url
    - sendPushNotification returns { success: true } on success
    - sendPushNotification with expired subscription (410 status) deletes subscription from DB and returns { success: false }
  </behavior>
  <action>
1. Install: `cd britv3.0 && pnpm add web-push && pnpm add -D @types/web-push`

2. Create `src/lib/push.ts`:
   - Import webpush from "web-push"
   - Lazy-init VAPID details on first call using VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_SUBJECT env vars
   - Export `getVapidPublicKey()`: returns `process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY`
   - Export `sendPushNotification(subscription: PushSubscriptionData, payload: { title: string; body: string; url?: string; icon?: string })`:
     - Call `webpush.sendNotification(subscription, JSON.stringify(payload))`
     - On 410 (Gone) error, delete subscription from push_subscriptions table using admin client, return `{ success: false, reason: "expired" }`
     - On success return `{ success: true }`
   - Export type `PushSubscriptionData = { endpoint: string; keys: { p256dh: string; auth: string } }`

3. Create `src/lib/push.test.ts`:
   - Mock web-push module
   - Test getVapidPublicKey returns env var
   - Test sendPushNotification calls webpush.sendNotification with correct payload
   - Test 410 error handling deletes subscription

4. Create `src/app/api/push/subscribe/route.ts`:
   - POST: Authenticate user, extract { endpoint, keys, userAgent } from body, upsert into push_subscriptions (UNIQUE on user_id + endpoint), return 201
   - DELETE: Authenticate user, extract { endpoint } from body, delete from push_subscriptions, return 200
   - Validate input with Zod schema

5. Create `src/app/api/push/send/route.ts`:
   - POST: Protected route (admin only or internal service call via service_role key)
   - Accept { userId, title, body, url }
   - Look up all push_subscriptions for userId
   - Call sendPushNotification for each subscription
   - Return { sent: number, failed: number }
  </action>
