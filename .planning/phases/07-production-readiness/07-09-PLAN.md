   - Deployment steps:

     3. Run migration CI (verify GitHub Action passes)
     4. Verify Sentry receiving events
     5. Verify PostHog receiving pageviews
     6. Smoke test: login, search, message, dashboard
   - Rollback procedure:
     1. Revert to previous Vercel deployment (Vercel dashboard -> Deployments -> Promote)
     2. If migration issue: document manual rollback SQL (ALTER TABLE, DROP statements)
     3. If env var issue: update in Vercel dashboard, redeploy
   - Post-launch monitoring: Sentry error rate, PostHog key metrics, Supabase dashboard (connection count, query performance)

   - PITR backup verification (LCH-12): steps to check Supabase PITR is enabled, verify backup schedule, document test restore procedure via Supabase dashboard

2. Create `docs/support-runbook.md` (1-page, per LCH-15):
   - Common issues and resolutions:
     - "User can't log in" -> Check auth logs in Supabase, verify email confirmed, check for suspended account
     - "Password reset email not received" -> Check Resend delivery logs, verify spam folder, check rate limit
     - "Listing not appearing in search" -> Check listing status (must be 'active'), verify geocoding completed, check materialized view refresh
     - "Push notifications not working" -> Check subscription in push_subscriptions table, verify VAPID keys, check browser permissions
     - "Provider verification stuck" -> Check admin queue, verify documents uploaded, contact admin
     - "Marketplace provider not found" -> Check service area overlap, verify provider profile active
     - "Maintenance request not updating" -> Check state machine transitions, verify user permissions
   - Escalation path: Check logs (Vercel) -> Check errors (Sentry) -> Check database (Supabase dashboard) -> Contact dev team
   - Legal compliance checklist (LCH-13):
     - GDPR: consent capture verified, data export working, deletion request flow tested, privacy policy up to date
     - Terms of Service: reviewed and current
     - Cookie policy: aligned with consent preferences
     - Property disclaimers: "This is not financial advice" on calculators, "Prices are indicative" on valuations
     - Data retention: documented retention periods per data type
