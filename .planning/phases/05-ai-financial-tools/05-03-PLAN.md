  <files>

  </files>
  <behavior>
    Recommendations:
    - getRecommendations returns properties matching saved search criteria (type, min/max price, bedrooms, location)
    - Excludes properties in user's viewed_properties, saved_properties, and dismissed_properties
    - Returns max 10 results ordered by match quality (most criteria matched)
    - Returns empty array when user has no saved searches
    - Returns empty array when no properties match

    Smart Replies:
    - getSuggestedReplies("viewing_request", "") returns viewing-related suggestions
    - getSuggestedReplies("quote_response", "") returns quote-related suggestions
    - getSuggestedReplies("general", "price") returns price-related suggestions (keyword match)
    - getSuggestedReplies returns max 4 suggestions
    - Unknown conversation type returns generic suggestions
  </behavior>
  <action>
    1. Create `src/services/recommendations/recommendations.ts`:
       - `getRecommendations(userId: string, limit?: number): Promise<PropertyRecommendation[]>`
       - Fetches user's saved searches from Supabase
       - Builds SQL query matching properties by: property_type, price range (min_price/max_price), bedrooms (>=), location (postcode prefix match)
       - Excludes properties where user has interactions (viewed, saved, dismissed) using NOT IN subquery
       - Orders by number of matching criteria descending
       - Limits to 10 results (configurable)
       - Uses Supabase server client for server-side data fetching
       - Returns PropertyRecommendation[] with property data + match_score

    2. Create `src/services/smart-replies/config.ts`:
       - Static config object mapping conversation types to reply suggestions:
         - "viewing_request": ["I'd love to arrange a viewing", "What times are available?", "Is the property still available?", "Can I bring someone along?"]
         - "quote_response": ["Thank you for the quote", "Can you break down the costs?", "When could you start?", "I'd like to proceed"]
         - "offer_discussion": ["I'd like to make an offer", "What's the asking price?", "Is there room for negotiation?", "Has there been much interest?"]
         - "maintenance": ["The issue is urgent", "When can someone visit?", "I've attached photos", "This has been ongoing"]
         - "general": generic fallback suggestions
       - Keyword map: { "price" => price-related suggestions, "urgent" => urgency suggestions, etc. }

    3. Create `src/services/smart-replies/smart-replies.ts`:
       - `getSuggestedReplies(conversationType: string, messageContent: string): string[]`
       - First checks conversation type for type-specific suggestions
       - Then scans messageContent for keywords to add relevant suggestions
       - Deduplicates and returns max 4 suggestions
       - Pure function, no API calls, no async

    4. Write tests for both services. Mock Supabase for recommendations.
  </action>
  <verify>
    <automated>cd britv3.0 && pnpm vitest run src/services/recommendations/ src/services/smart-replies/</automated>
  </verify>
  <done>Recommendations service queries by saved search criteria with exclusions, smart replies return config-driven suggestions by type and keywords, all tests pass</done>

    2. Create `src/services/land-registry/land-registry.ts`:
       - `getPricePaidData(postcode: string, radius?: number): Promise<PricePaidRecord[]>`
         Queries price_paid_data table by postcode prefix (outward code, e.g. "SW1A")
         Returns last 20 transactions sorted by date descending
         Uses Supabase server client
       - `getAreaPriceTrend(postcode: string, years?: number): Promise<AreaPriceTrend[]>`
         Groups price_paid_data by year for the postcode area
         Returns average price and transaction count per year for last 5 years (configurable)

       - `getPricePaidSummary(postcode: string): Promise<PricePaidSummary>`
         Combines recent sales + area trend + current average
