**Review service (src/services/marketplace/review-service.ts):**

1. `createReview(supabase, userId, data: ReviewCreateInput & { booking_id: string })` -- Validate with reviewCreateSchema. Verify booking exists, status='completed', and user_id matches. The UNIQUE constraint on booking_id prevents duplicate reviews at DB level. Run `analyzeReviewSentiment(data.review_text)` to get sentiment + confidence. Run `detectSpam(data.review_text)` to get spam indicators. INSERT into reviews with: ratings, text, sentiment result, spam_indicators JSONB, moderation_status='pending'. The DB trigger `on_review_created` will calculate authenticity_score. After insert, INSERT into moderation_queue with priority_score calculated from: spam_score * 3 + (fake_review_probability > 0.7 ? 10 : 0) + (extreme rating deviation ? 1 : 0). Return created review.

2. `listProviderReviews(supabase, providerId, options: { sort?: "recent" | "helpful" | "relevant"; minRating?: number; maxRating?: number; searchQuery?: string; limit?: number; offset?: number })` -- SELECT approved reviews for provider. Sort by created_at DESC (recent), helpful_count DESC (helpful), or full-text search rank (relevant if searchQuery provided). Filter by rating range. Include provider_response if exists. Return paginated results.

3. `voteHelpfulness(supabase, userId, reviewId, isHelpful: boolean)` -- UPSERT into review_helpfulness (update if user already voted). Update helpful_count/not_helpful_count on reviews table. Return updated counts.

4. `respondToReview(supabase, providerId, reviewId, responseText: string)` -- Verify provider owns the review's service. Validate response length (1-1000 chars). UPDATE reviews SET provider_response, provider_response_at. One response per review (if provider_response already set, update it). Return updated review.

5. `flagReview(supabase, userId, reviewId, reason: ReviewFlagReason, description?: string)` -- Validate with reviewFlagSchema. Verify user is not the reviewer (can't flag own review). INSERT into review_flags. Increment flag_count on reviews. If flag_count >= 3, update moderation_queue priority. Return flag record.

**Moderation service (src/services/marketplace/moderation-service.ts):**

1. `getModerationQueue(supabase, options: { status?: string; limit?: number; offset?: number })` -- SELECT from moderation_queue JOIN reviews. Only accessible by admins (RLS enforced). Ordered by priority_score DESC, created_at ASC. Include review details, flag count, spam indicators.

2. `moderateReview(supabase, moderatorId, queueEntryId, decision: "approve" | "reject" | "flag_for_review", reason?: string)` -- UPDATE moderation_queue with decision, reason, completed_at, assigned_to. UPDATE review moderation_status accordingly. If approved: the DB trigger `update_provider_rating_stats_incremental` fires. If rejected: notify reviewer (insert notification if table exists). Return updated entry.

**Review routes:**

1. `POST /api/reviews/create` -- Auth required. Parse JSON body with booking_id. Call createReview. Return 201 or 409 if review already exists for booking (unique constraint).

2. `GET /api/reviews/list` -- Public for approved reviews. Parse query params: provider_id (required), sort, min_rating, max_rating, search_query, limit, offset. Call listProviderReviews. Return 200 with paginated results including rating stats from provider_rating_stats table.

3. `POST /api/reviews/[id]/helpful` -- Auth required. Parse JSON body: { is_helpful: boolean }. Call voteHelpfulness. Return 200 with updated counts.

4. `POST /api/reviews/[id]/flag` -- Auth required. Parse JSON body: { reason, description }. Call flagReview. Return 201 or 403 if flagging own review.

5. `POST /api/reviews/[id]/respond` -- Auth required (provider). Parse JSON body: { response: string }. Call respondToReview. Return 200 or 403 if not the provider.

6. `PATCH /api/reviews/moderation/[id]` -- Auth required (admin only). Parse JSON body: { decision, reason }. Call moderateReview. Return 200 or 403 if not admin.

All routes: structured JSON responses, appropriate status codes, try/catch error handling.
