**File validator (src/lib/marketplace/file-validator.ts):**
- Import `fileTypeFromBuffer` from "file-type" (ESM import)
- Export `ALLOWED_TYPES = new Set(["application/pdf", "image/jpeg", "image/png", "image/webp"])`
- Export `MAX_FILE_SIZE = 10 * 1024 * 1024`
- Export `async function validateFile(buffer: Buffer): Promise<{ mime: string; ext: string }>` -- validates magic bytes and size, throws descriptive errors
- Note: file-type is ESM-only, use dynamic import if needed for compatibility

**Booking state machine (src/lib/marketplace/booking-state-machine.ts):**
- Export `BookingStatus` type (6 values)
- Export `TransitionActor` type ("user" | "provider" | "system")
- Export `VALID_TRANSITIONS` array with all 9 transitions from epic spec S06 (pending_confirmation->confirmed by provider, etc.)
- Export `canTransition(currentStatus, newStatus, actorRole): { allowed: boolean; requiresReason: boolean }`
- Export `getValidNextStatuses(currentStatus, actorRole): BookingStatus[]` -- lists what statuses can be transitioned to

**Sentiment analyzer (src/lib/marketplace/sentiment-analyzer.ts):**
- Export `SentimentScore` type (5 values)
- Export `analyzeReviewSentiment(text: string): { sentiment: SentimentScore; confidence: number }`
- Implement exactly as in epic spec Section 5.6: POSITIVE_WORDS (25+), NEGATIVE_WORDS (25+), INTENSIFIERS, normalized scoring
- Confidence = min(matchedWords / 5, 1)

**Spam detector (src/lib/marketplace/spam-detector.ts):**
- Export `SpamIndicators` type: { has_contact_info: boolean; has_links: boolean; has_excessive_caps: boolean; has_promotional: boolean; has_repeated_chars: boolean; has_excessive_punctuation: boolean; spam_score: number }
- Export `detectSpam(text: string): SpamIndicators`
- Detect: phone numbers (UK format regex), email addresses, URLs, ALL CAPS ratio >50%, promotional keywords ("discount", "free", "click here", "call now"), repeated chars (3+ same char), excessive punctuation (>5 consecutive)
- spam_score = sum of individual boolean flags (0-6 scale)
