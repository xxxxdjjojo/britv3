**RFQ service (src/services/marketplace/rfq-service.ts):**

1. `createRfq(supabase, userId, data: RfqCreateInput)` -- Validate with rfqCreateSchema. Geocode property_postcode via postcodes-io to get lat/lng. INSERT into service_requests with status='open', expires_at = now() + 30 days. After insert, call `inngest.send({ name: "marketplace/rfq.created", data: { rfqId } })` for async provider matching. Returns the created RFQ.

2. `getRfq(supabase, rfqId: string)` -- SELECT with JOIN to quotes (count). RLS handles access.

3. `listUserRfqs(supabase, userId, status?: RfqStatus, limit=20, offset=0)` -- SELECT where user_id matches, ordered by created_at DESC. Returns paginated results.

4. `listProviderMatchedRfqs(supabase, providerId, limit=20, offset=0)` -- SELECT open RFQs matching provider's service categories. RLS + service_category overlap check. Ordered by created_at DESC.

5. `matchProvidersForRfq(supabase, rfqId: string)` -- Internal function used by Inngest. Loads the RFQ, then calls Supabase RPC function `get_matching_providers_for_rfq` or equivalent logic: score by category match (50pts), postcode overlap (30pts), proximity (20pts), rating bonus (10pts). Return top 10 providers. Note: Create the RPC function in migration if not already there, OR implement matching logic in TypeScript calling search_providers with the RFQ's category and location.

**Quote service (src/services/marketplace/quote-service.ts):**

1. `createQuote(supabase, providerId, data: QuoteCreateInput & { service_request_id: string })` -- Validate with quoteCreateSchema. Check provider is verified. Check no existing active quote from this provider for this RFQ. INSERT with status='sent'. Quote number auto-generated by DB trigger. Increment RFQ quote_count. Update RFQ status to 'quotes_received' if first quote. Returns created quote.

2. `getQuote(supabase, quoteId: string)` -- SELECT with provider details.

3. `listQuotesForRfq(supabase, rfqId: string)` -- SELECT all non-withdrawn quotes for an RFQ, ordered by total_amount. RLS ensures only RFQ owner or quote provider can see.

4. `acceptQuote(supabase, userId, quoteId: string)` -- Verify user owns the RFQ. UPDATE quote status to 'accepted'. The partial unique index `(service_request_id WHERE status='accepted')` prevents race conditions -- if another quote was already accepted, this UPDATE will fail with a unique constraint violation. On success, UPDATE all other quotes for this RFQ to 'declined'. UPDATE RFQ status to 'awarded'. Returns the accepted quote. Wrap in try/catch for unique constraint errors.

5. `declineQuote(supabase, userId, quoteId: string, reason?: string)` -- UPDATE quote status to 'declined' with optional decline reason.

**Inngest notification function (src/inngest/functions/rfq-notify-providers.ts):**
- Export `rfqNotifyProviders` created with inngest.createFunction
- Trigger: event "marketplace/rfq.created"
- Step 1 ("get-matching-providers"): Call matchProvidersForRfq to get up to 10 providers
- Step 2 ("send-in-app-notifications"): INSERT notification rows for each matched provider (assumes a notifications table exists from Phase 3; if not, skip this step and log)
- Step 3 ("wait-for-in-app-view"): step.sleep("1h")
- Step 4 ("send-email-fallback"): Check which notifications are still unread, send email via Resend for unread ones (or log if Resend not configured)

**Update Inngest serve endpoint** to include rfqNotifyProviders in the functions array.

**RFQ routes:**

1. `POST /api/rfq/create` -- Auth required. Parse JSON body, validate with rfqCreateSchema. Call createRfq. Return 201 with RFQ data.

2. `GET /api/rfq/[id]` -- Auth required. Get RFQ by ID. Return 200 or 404. RLS handles authorization.

3. `GET /api/rfq/list` -- Auth required. Parse query params: status, limit, offset, role ("user" or "provider"). If role=user, call listUserRfqs. If role=provider, call listProviderMatchedRfqs. Return 200 with paginated results.

**Quote routes:**

4. `POST /api/quotes/create` -- Auth required (provider only). Parse JSON body with service_request_id. Validate with quoteCreateSchema. Call createQuote. Return 201 with quote data or 403 if not verified.

5. `GET /api/quotes/[id]` -- Auth required. Get quote by ID with details. Return 200 or 404.

6. `POST /api/quotes/[id]/accept` -- Auth required (RFQ owner only). Call acceptQuote. Return 200 on success, 409 if another quote already accepted (unique constraint violation), 403 if not RFQ owner.

All routes: use createServerClient for Supabase, validate auth, return structured JSON errors, appropriate HTTP status codes.
