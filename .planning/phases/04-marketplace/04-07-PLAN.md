**Components (all "use client" where needed for interactivity):**

1. **SearchFilters.tsx** -- Service category dropdown (16 options), postcode input, radius slider (1-100 miles, default 25), minimum rating selector (1-5 stars), free-text search input. Uses controlled state. Calls onChange callback with filter params. Use Shadcn Select, Input, Slider components.

2. **ProviderCard.tsx** -- Displays: business_name, services badges (max 3 shown), average_rating with RatingStars, review_count, distance_miles, years_in_business. Link to /marketplace/[slug]. Use Shadcn Card component.

3. **RFQCreateForm.tsx** -- Full form using react-hook-form + Zod resolver with rfqCreateSchema. Fields: service_category select, title input, description textarea, property_postcode input (with UK format hint), preferred_start_date date picker, urgency radio (low/medium/high/urgent), budget_min and budget_max number inputs, file attachments (up to 5). Submit handler calls POST /api/rfq/create. Shows success state with RFQ reference. Use Shadcn Form components.

4. **QuoteCreateForm.tsx** -- For providers. react-hook-form with quoteCreateSchema. Dynamic line items array (add/remove rows): description, quantity, unit_price, auto-calculated total. Scope of work textarea. Estimated duration. Payment terms. Warranty info. Validity date. VAT toggle. Auto-calculated grand total. Submit to POST /api/quotes/create.

5. **QuoteComparison.tsx** -- Accepts up to 3 quote objects. Displays side-by-side in columns. Shows: provider name, rating, total amount, VAT status, line items table, scope summary, duration, warranty, payment terms. Highlights cheapest/most expensive. "Accept" button per quote. Uses Shadcn Table and Card.

6. **RatingStars.tsx** -- Renders 1-5 stars (filled/empty/half). Accepts rating number and optional size. Uses lucide-react Star icon.

7. **RatingDistribution.tsx** -- Horizontal bar chart showing 5-star to 1-star distribution. Accepts counts object { count_5_star, count_4_star, ... }. Each bar shows star label, bar width proportional to count, count number.

8. **ReviewsList.tsx** -- Paginated list of reviews. Each review shows: RatingStars, title, text (3-line clamp with "Read more"), helpful count, date, provider response (if exists). Helpfulness voting buttons (thumbs up/down). Flag button. Pagination at bottom. Sort selector (recent/helpful).

**Pages:**

9. **marketplace/page.tsx** -- Server Component. Renders SearchFilters and grid of ProviderCards. Uses URL search params for filter state. Fetches from /api/providers/search on the server side. Shows result count and "No providers found" empty state.

10. **marketplace/[slug]/page.tsx** -- Server Component. Fetches provider by slug. Shows full profile: business info, services list, coverage area, pricing info, qualifications. Shows RatingDistribution and ReviewsList. "Request Quote" button links to RFQ create with pre-selected category.

All components use Tailwind classes consistent with existing globals.css theme. Responsive: grid cols reduce on mobile (lg:3 -> md:2 -> 1).

**Components:**

1. **BookingTimeline.tsx** -- Vertical timeline showing booking_status_history entries. Each entry: status badge, timestamp, changed_by name, reason (if provided). Current status highlighted. Uses Shadcn Badge for status colors.

2. **BookingStatusBadge.tsx** -- Colored badge for booking status. Colors: pending_confirmation=yellow, confirmed=blue, in_progress=orange, completed=green, cancelled=red, disputed=purple. Uses Shadcn Badge with variant.

3. **ReviewForm.tsx** -- "use client". react-hook-form with reviewCreateSchema. Star rating inputs for: overall (required), punctuality, quality, value, professionalism (all optional but encouraged). Title input. Review text textarea (20-2000 chars with counter). Submit to POST /api/reviews/create. Success state shows review pending moderation message.

4. **ProviderProfileForm.tsx** -- "use client". react-hook-form with providerProfileSchema. Business name, description textarea, services multi-select (16 categories), service postcodes input (comma-separated or tag input), service_radius slider, pricing fields (call_out_fee, hourly_rate, day_rate as optional numbers), qualifications/accreditations textarea. Submit to POST/PUT provider profile API.

5. **DocumentUpload.tsx** -- "use client". Drag-and-drop file upload zone. Shows allowed types (PDF, JPEG, PNG, WebP) and 10MB limit. File list with type selector dropdown (6 document types). Upload progress indicator. Status badges (pending, approved, rejected). Calls POST /api/providers/documents/upload.

6. **AvailabilityCalendar.tsx** -- "use client". Simple date range picker for marking unavailable periods. Shows existing unavailable periods and confirmed bookings. Uses Shadcn Calendar component for date selection. "Add unavailable period" form with start/end dates and optional reason.

**Dashboard Pages (all "use client" or Server Component as appropriate):**

7. **rfqs/page.tsx** -- Lists user's RFQs with status badges. Filter by status. Each row: title, category, quote_count, status, created date. Link to detail page.

8. **rfqs/create/page.tsx** -- Renders RFQCreateForm component.

9. **rfqs/[id]/page.tsx** -- RFQ detail with all info. Below: list of received quotes with QuoteComparison component. "Accept Quote" action buttons.

10. **bookings/page.tsx** -- Lists bookings (user view or provider view based on role). Status filter tabs. Stats cards: pending/confirmed/in_progress/completed counts. Each row: reference, service title, other party name, dates, amount, BookingStatusBadge. Quick-action buttons (confirm for provider, cancel for either).

11. **bookings/[id]/page.tsx** -- Full booking detail. BookingTimeline component. Status transition buttons based on current status and user role (uses getValidNextStatuses logic). Booking info card. Link to "Leave Review" for completed bookings.

12. **provider/profile/page.tsx** -- Renders ProviderProfileForm. Shows verification status badge.

13. **provider/documents/page.tsx** -- Renders DocumentUpload. Shows existing documents with status.

14. **provider/quotes/page.tsx** -- Lists provider's submitted quotes. Filter by status. Each row: quote number, RFQ title, amount, status, date. Link to QuoteCreateForm for new quotes.

15. **provider/availability/page.tsx** -- Renders AvailabilityCalendar.

16. **reviews/page.tsx** -- Shows reviews the user has written AND reviews received (for providers). ReviewForm accessible from here for eligible bookings.

All pages use Tailwind, follow responsive patterns, use Shadcn components for consistency.
