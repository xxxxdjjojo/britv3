**agent-crm-service.ts:**
- `getCrmClients(supabase, agentId, filters?: { client_type?, search?, tags?, page?, limit? })` — queries agent_crm_clients with optional filters, supports text search on name/email, pagination
- `getCrmClientById(supabase, clientId, agentId)` — single client with linked properties (join with listings where buyer/seller matches), communication history (join with messages)
- `createCrmClient(supabase, agentId, input: CreateCrmClientInput)` — validates with Zod, inserts
- `updateCrmClient(supabase, clientId, agentId, input)` — updates client, sets last_contact_at if communication
- `searchCrmClients(supabase, agentId, query)` — full-text search on name, email, phone, notes

**agent-team-service.ts:**
- `getTeamMembers(supabase, agentId, branchId?)` — queries agent_team_members with optional branch filter, includes performance metrics (lead count, viewings count)
- `inviteTeamMember(supabase, agentId, input: CreateTeamMemberInput)` — inserts team member with 'pending' status, sends invite email via Resend to input.email with agency name
- `updateTeamMemberRole(supabase, memberId, agentId, newRole: TeamRole)` — updates role, validates against TEAM_ROLES
- `removeTeamMember(supabase, memberId, agentId)` — sets status to 'inactive'
- `getBranches(supabase, agentId)` — all branches with team member count
- `createBranch(supabase, agentId, input: CreateBranchInput)` — inserts branch
- `updateBranch(supabase, branchId, agentId, input)` — updates branch details
- `assignMemberToBranch(supabase, memberId, agentId, branchId)` — updates branch_id on team member

**agent-analytics-service.ts:**
- `getAgentPerformanceReport(supabase, agentId, dateRange?)` — aggregates: listings sold count, avg time-on-market, total revenue (from commissions), conversion rate (leads closed / total), client satisfaction (from reviews avg)
- `getBranchPerformanceReport(supabase, agentId, branchId)` — same metrics scoped to branch via team member IDs
- `getCompetitorAnalysis(supabase, agentId, area)` — queries listings table grouped by agent, computes: count, avg price, avg time-on-market for agents in same postcode district. Area derived from agent's agency_profiles.coverage_areas.
- `generateVendorReport(supabase, agentId, propertyId, reportType)` — aggregates property performance data (views, viewings, offers, market comparison) into agent_vendor_reports table, returns report data for PDF
- `getMarketAppraisalData(supabase, postcode)` — queries listings and price_history for comparable properties within same postcode district, calculates suggested price range with confidence band

**agent-billing-service.ts:**
- `createCheckoutSession(agentId, priceId, successUrl, cancelUrl)` — creates Stripe Checkout session for subscription. Uses `new Stripe(process.env.STRIPE_SECRET_KEY!)`. Returns session URL.
- `getCustomerPortalUrl(agentId)` — creates Stripe billing portal session. Returns portal URL.
- `getCurrentSubscription(supabase, agentId)` — queries profiles for stripe_customer_id, retrieves subscription from Stripe API
- `purchaseFeatureBoost(agentId, listingId, durationDays, priceId)` — creates Stripe Checkout for one-time featured listing boost. On success webhook sets featured_until on listing.
- `generateApiKey(supabase, agentId, name)` — generates random API key (crypto.randomUUID), stores hash (crypto.createHash('sha256')), stores prefix (first 8 chars), returns the full key ONCE
- `revokeApiKey(supabase, keyId, agentId)` — sets is_active=false, sets revoked_at
- `getApiKeys(supabase, agentId)` — returns all keys (prefix only, not hash) with usage stats

**agent-feed-service.ts:**
- `getFeedIntegrations(supabase, agentId)` — all feed configs with sync status
- `createFeedIntegration(supabase, agentId, input: { provider, api_key, field_mapping })` — inserts with 'disconnected' status. NOTE: api_key is stored encrypted (use Supabase vault or simple AES encryption via Node crypto)
- `updateFeedIntegration(supabase, integrationId, agentId, input)` — updates config
- `deleteFeedIntegration(supabase, integrationId, agentId)` — hard delete
- `getFeedSyncStatus(supabase, integrationId, agentId)` — returns sync_status, last_sync_at, error_log

**API routes** (all follow same pattern: auth check, validate, call service, return JSON):
- `api/agent/crm/route.ts`: GET (list+search), POST (create), PATCH (update)
- `api/agent/team/route.ts`: GET (members+branches), POST (invite member, create branch), PATCH (update role, assign branch), DELETE (remove member)
- `api/agent/analytics/route.ts`: GET with ?type=agent|branch|competitor|appraisal query param
- `api/agent/billing/route.ts`: GET (subscription), POST (checkout session or portal URL based on ?action param)
- `api/agent/feeds/route.ts`: GET (list), POST (create), PATCH (update), DELETE (remove)
- `api/agent/reports/route.ts`: POST (generate vendor report), GET (list reports for property)
