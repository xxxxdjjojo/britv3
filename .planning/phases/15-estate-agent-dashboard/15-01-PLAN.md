<task type="auto">
  <name>Task 1: Create Supabase migration for agent dashboard tables</name>
  <files>supabase/migrations/20260313_agent_dashboard.sql</files>
  <action>
Create a single SQL migration file with the following tables (all with RLS enabled):

1. **agent_agency_profiles** — Agency profile/branding data: id (PK), agent_id (FK profiles, UNIQUE), agency_name, contact_email, contact_phone, address_line_1, address_line_2, city, postcode, description, specializations (TEXT[]), coverage_areas (TEXT[]), logo_url, brand_primary_colour, brand_secondary_colour, social_facebook, social_twitter, social_instagram, social_linkedin, website_url, created_at, updated_at. RLS: agent_id = auth.uid().

2. **agent_leads** — Lead pipeline: id (PK), agent_id (FK profiles), property_id (FK listings, nullable), contact_name, contact_email, contact_phone, stage TEXT CHECK ('new_enquiry','qualified','viewing_booked','offer_made','closed'), source TEXT CHECK ('website','portal','phone','walk_in','referral','other'), assigned_to (FK profiles, nullable), notes, created_at, updated_at. RLS: agent_id = auth.uid() OR auth.uid() IN team members.

3. **agent_lead_activities** — Lead timeline: id (PK), lead_id (FK agent_leads), actor_id (FK profiles), activity_type TEXT, description, metadata JSONB, created_at. RLS: via lead ownership.

4. **agent_offers** — Offers: id (PK), agent_id (FK profiles), property_id (FK listings), lead_id (FK agent_leads, nullable), buyer_name, buyer_email, buyer_phone, amount BIGINT (pence), conditions TEXT, solicitor_details JSONB, aip_status TEXT CHECK ('not_provided','provided','verified'), status TEXT CHECK ('pending','accepted','rejected','countered','withdrawn'), counter_amount BIGINT (nullable), vendor_notified BOOLEAN DEFAULT false, created_at, updated_at. RLS: agent_id ownership or team.

5. **agent_offer_history** — Offer audit trail: id (PK), offer_id (FK agent_offers), previous_status, new_status, actor_id (FK profiles), note, created_at. RLS: via offer ownership.

6. **agent_sale_progressions** — Sale tracking: id (PK), agent_id (FK profiles), offer_id (FK agent_offers, UNIQUE), property_id (FK listings), stage TEXT CHECK (8 UK conveyancing stages), expected_completion_date DATE, solicitor_buyer JSONB, solicitor_seller JSONB, notes, created_at, updated_at. RLS: agent_id ownership.

7. **agent_commissions** — Commission tracking: id (PK), agent_id (FK profiles), property_id (FK listings), sale_price BIGINT, commission_rate DECIMAL(5,2), commission_amount BIGINT, status TEXT CHECK ('pending','invoiced','paid'), paid_at TIMESTAMPTZ, created_at. RLS: agent_id ownership.

8. **agent_team_members** — Team: id (PK), agent_id (FK profiles), user_id (FK profiles), branch_id (FK agent_branches, nullable), role TEXT CHECK ('admin','senior_negotiator','negotiator','lettings_manager','viewer'), status TEXT CHECK ('active','inactive','pending'), email TEXT, name TEXT, invited_at, joined_at, created_at. RLS: agent_id ownership. UNIQUE(agent_id, user_id).

9. **agent_branches** — Branches: id (PK), agent_id (FK profiles), name, address_line_1, address_line_2, city, postcode, phone, email, is_head_office BOOLEAN DEFAULT false, created_at, updated_at. RLS: agent_id ownership.

10. **agent_crm_clients** — CRM: id (PK), agent_id (FK profiles), user_id (FK profiles, nullable), name, email, phone, client_type TEXT CHECK ('buyer','seller','landlord','tenant'), preferences JSONB, notes, tags TEXT[], last_contact_at, created_at, updated_at. RLS: agent_id ownership.

11. **agent_viewing_slots** — Availability: id (PK), agent_id (FK profiles), property_id (FK listings), start_time TIMESTAMPTZ, end_time TIMESTAMPTZ, is_booked BOOLEAN DEFAULT false, booked_by (FK profiles, nullable), notes, created_at. RLS: agent_id ownership; buyers can SELECT booked slots.

12. **agent_viewing_feedback** — Post-viewing: id (PK), agent_id (FK profiles), viewing_slot_id (FK agent_viewing_slots), buyer_name, interest_level INT CHECK (1-5), price_opinion TEXT CHECK ('too_high','about_right','good_value'), likelihood_to_offer TEXT CHECK ('unlikely','possible','likely','very_likely'), comments, created_at. RLS: agent_id ownership.

13. **agent_api_keys** — API keys: id (PK), agent_id (FK profiles), key_hash TEXT (store hash only), key_prefix TEXT (first 8 chars for display), name, rate_limit_per_minute INT DEFAULT 60, last_used_at, usage_count BIGINT DEFAULT 0, is_active BOOLEAN DEFAULT true, created_at, revoked_at. RLS: agent_id ownership.

14. **agent_feed_integrations** — Property feeds: id (PK), agent_id (FK profiles), provider TEXT CHECK ('reapit','alto','jupix'), api_key_encrypted TEXT, webhook_url TEXT, sync_status TEXT CHECK ('disconnected','connected','syncing','error'), last_sync_at, field_mapping JSONB, error_log JSONB[], created_at, updated_at. RLS: agent_id ownership.

15. **agent_vendor_reports** — Generated reports: id (PK), agent_id (FK profiles), property_id (FK listings), report_type TEXT CHECK ('listing_performance','viewing_summary','market_analysis'), data JSONB, generated_at, pdf_url TEXT. RLS: agent_id ownership.

Also create:
- Indexes on agent_id for all tables, composite indexes on (agent_id, stage) for leads, (agent_id, status) for offers
- Updated_at trigger function (reuse if exists) on tables with updated_at
- RPC function `get_agent_dashboard_kpis(p_agent_id UUID)` returning active_listings_count, new_leads_count, viewings_this_week_count, pending_offers_count, performance_score (computed as closed_leads/total_leads ratio)

Use the established pattern from existing migrations (landlord, seller). All monetary values in pence (BIGINT).
