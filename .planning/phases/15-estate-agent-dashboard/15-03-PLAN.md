  <files>britv3.0/src/services/agent/agent-viewing-service.ts, britv3.0/src/services/agent/agent-offer-service.ts, britv3.0/src/services/agent/agent-sale-service.ts</files>
  <action>
**agent-viewing-service.ts:**
- `getAgentViewingSlots(supabase, agentId, propertyId?, dateRange?)` — queries agent_viewing_slots, optional filter by property and date range (start_time between), includes booked_by profile name
- `createViewingSlot(supabase, agentId, input: { property_id, start_time, end_time, notes? })` — inserts slot, validates end_time > start_time
- `updateViewingSlot(supabase, slotId, agentId, input)` — updates slot, only if not yet booked
- `deleteViewingSlot(supabase, slotId, agentId)` — deletes slot, only if not booked
- `getViewingFeedback(supabase, agentId, propertyId?)` — queries agent_viewing_feedback with optional property filter
- `submitViewingFeedback(supabase, agentId, input: { viewing_slot_id, buyer_name, interest_level, price_opinion, likelihood_to_offer, comments? })` — inserts feedback record

**agent-offer-service.ts:**
- `getAgentOffers(supabase, agentId, propertyId?, status?)` — queries agent_offers grouped by property_id using Supabase select, includes buyer details and AIP status
- `getOfferById(supabase, offerId, agentId)` — single offer with full history from agent_offer_history
- `createOffer(supabase, agentId, input: CreateOfferInput)` — inserts offer with 'pending' status, creates initial history entry
- `updateOfferStatus(supabase, offerId, agentId, newStatus, note?)` — updates status, inserts history entry with previous/new status, sets vendor_notified=true. If status='accepted', call createSaleProgression from agent-sale-service to initialize the sale pipeline.
- `counterOffer(supabase, offerId, agentId, counterAmount, note?)` — updates status to 'countered', sets counter_amount, creates history
- `getOfferHistory(supabase, offerId)` — returns all history entries ordered by created_at

**agent-sale-service.ts:**
- Define `ALLOWED_TRANSITIONS` map: each stage maps to [next_stage, previous_stage] to allow forward progression and one-step rollback
- `getAgentSaleProgressions(supabase, agentId)` — queries all active sale progressions with property and offer details, for Kanban board
- `createSaleProgression(supabase, agentId, offerId, propertyId)` — creates progression at 'offer_accepted' stage
- `updateSaleStage(supabase, progressionId, agentId, newStage, notes?)` — validates transition against ALLOWED_TRANSITIONS, updates stage, updates updated_at. Throws error if invalid transition.
- `getSaleProgressionById(supabase, progressionId, agentId)` — single progression with offer and property details

All functions use SupabaseClient pattern, throw on errors, import types from @/types/agent.
