**offers/page.tsx** (Server Component):
- Fetch offers via getAgentOffers(supabase, user.id)
- Group by property_id
- Pass grouped offers to OffersDashboard

**OffersDashboard.tsx** ("use client"):
- Render offers grouped by property: each property section shows address, listing image, and a list of offers underneath
- Each offer card shows: buyer name, amount (formatted as GBP), AIP status badge (not_provided=grey, provided=yellow, verified=green), offer status badge (pending=blue, accepted=green, rejected=red, countered=orange, withdrawn=grey), conditions summary, date
- Filter bar: filter by status, search by buyer name
- Sort: by amount (high-low), date (newest), status
- Click on offer navigates to negotiation thread
- Supabase Realtime subscription for new offers

**offers/[id]/page.tsx** (Server Component):
- Fetch offer by ID via getOfferById(supabase, params.id, user.id)
- Fetch offer history via getOfferHistory(supabase, params.id)
- Pass to NegotiationThread

**NegotiationThread.tsx** ("use client"):
- Top: Offer summary card (buyer details, amount, property, conditions, solicitor details, AIP document link if provided)
- Middle: Timeline of offer history (status changes with timestamps, actor names, notes)
- Bottom: Action buttons based on current status:
  - If pending: "Accept" (green), "Reject" (red), "Counter" (orange -- opens counter amount input dialog)
  - If countered: "Accept Counter" or "Reject"
  - If accepted/rejected/withdrawn: show final status, no actions
- Accept: calls updateOfferStatus with 'accepted', which also creates a sale progression
- Counter: shows Shadcn Dialog with counter amount input and optional note
- Reject: confirmation dialog with optional reason
- "Notify Vendor" toggle/button: marks vendor_notified=true
- All actions show sonner toast on completion
