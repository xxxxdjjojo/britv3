   - `getFinancialEntries(supabase, propertyId, filters?: {type?, category?, dateRange?})`: Query financial_entries for property. Support filtering by type (income/expense), category, and date range. Order by entry_date DESC.
   - `createFinancialEntry(supabase, propertyId, data)`: Validate with financialEntrySchema. Insert into financial_entries. Set user_id from auth.uid(). If type='income' and category='rent', also set tenancy_id, rent_period_start, rent_period_end.
   - `getFinancialSummary(supabase, propertyId, startDate, endDate)`: Call Supabase RPC `get_property_financial_summary` with property_id, start_date, end_date. Returns {total_income, total_expenses, net_income, entry_count}.
   - `uploadReceipt(supabase, propertyId, entryId, file: File)`: Validate file type (PDF, JPG, PNG only via validateFileType). If image, compress via compressReceipt(). Upload to Supabase Storage bucket 'expense-receipts' at path `{propertyId}/{entryId}/{filename}`. Update receipt_url on the financial_entry record.
   - `getRentPaymentsForTenancy(supabase, tenancyId)`: Query financial_entries where tenancy_id matches and type='income' and category='rent'. For use by RentStatusIndicator.

2. Create API route `GET /api/properties/[id]/financials`:
   - Query params: type, category, start_date, end_date, summary_only (boolean).
   - If summary_only=true: call getFinancialSummary, return summary object.
   - Otherwise: call getFinancialEntries with filters, return entries array.

   Create API route `POST /api/properties/[id]/financials`:
   - Validate body with financialEntrySchema. Call createFinancialEntry. Return 201 with created entry.
   - Receipt file upload handled separately via client-side Supabase Storage upload.

   Period presets for summary:
   - "this_month": start of current month to today
   - "this_quarter": start of current quarter to today
   - "ytd": start of year to today
   - "last_12_months": today minus 12 months to today
