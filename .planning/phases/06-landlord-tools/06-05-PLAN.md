  <action>
1. Create `britv3.0/src/services/landlord/document-service.ts`:
   - `getDocuments(supabase, propertyId, filters?: {category?})`: Query property_documents for property, ordered by created_at DESC. Include computed expiry status.
   - `createDocument(supabase, propertyId, metadata, file: File)`: Validate file type via validateFileType() (reject non-PDF/JPG/PNG). Check file size (max 2MB). If image, compress via compressImage(). Upload to Supabase Storage 'property-documents' at `{propertyId}/{documentId}/{filename}`. Insert metadata into property_documents table. The DB trigger auto-calculates next_reminder_date from expiry_date.
   - `deleteDocument(supabase, documentId)`: Delete from property_documents table AND from Supabase Storage. Fetch the record first to get file_url for storage deletion.
   - `getExpiringDocuments(supabase, propertyId)`: Query documents where expiry_date is within 30 days of today. Used by ComplianceAlert.

2. Create API routes:
   - `GET /api/properties/[id]/documents` -- optional category filter. Returns documents array.
   - `POST /api/properties/[id]/documents` -- validates body with documentUploadSchema. File upload handled client-side to Storage, metadata posted here. Returns 201.
   - `DELETE /api/documents/[id]` -- deletes document record and storage file. Returns 204.

3. Create `DocumentUpload.tsx` ("use client"):
   - Form with: name (text input), category (select from document_category enum: Gas Safety, Electrical EICR, EPC, Insurance, etc.), expiry_date (optional date input), file (file input accepting PDF/JPG/PNG).
   - On file select: validate via validateFileType(). If invalid, show error toast "Only PDF, JPG, and PNG files are allowed." Check size <= 2MB. If image, compress. Show file name and size.
   - Optional: link to specific tenancy (select dropdown of tenancies).
   - Submit: upload file to Storage via client Supabase SDK, then POST metadata to API.
   - Use React Hook Form + documentUploadSchema + sonner toast.

4. Create `DocumentList.tsx` (Server Component):
   - Receives documents array. Renders table: name, category, upload date, expiry date, expiry status indicator, actions (download, delete).
   - Expiry status: green dot = valid (expiry > 30 days away), amber dot = expiring within 30 days, red dot = expired, no dot = no expiry date set.
   - Download: generate signed URL via Supabase Storage. Delete: call DELETE /api/documents/[id] with confirmation dialog.

5. Create `ComplianceAlert.tsx` (Server Component):
   - Receives expiring documents count and list. If count > 0, render amber banner: "{N} compliance document(s) expiring soon" with link to documents tab.
   - Used on property overview page and portfolio cards.

6. Create `documents/page.tsx`: Server Component. Fetch documents for property. Layout: ComplianceAlert at top (if any expiring), DocumentUpload form, DocumentList below.

7. Write tests:
   - `document-upload.test.ts`: Test file validation (accept PDF/JPG/PNG, reject EXE/TXT), size limit enforcement, form field validation.
   - `compliance-reminders.test.ts`: Test reminder date calculation (expiry - 30 days), expiry status indicator logic (green/amber/red), ComplianceAlert renders when documents expiring.
