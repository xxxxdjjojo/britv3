  <files>britv3.0/supabase/functions/compliance-reminders/index.ts, britv3.0/supabase/migrations/20260307_epic7_compliance_cron.sql, britv3.0/src/__tests__/landlord/compliance-edge-function.test.ts</files>
  <action>
1. Create Supabase Edge Function at `britv3.0/supabase/functions/compliance-reminders/index.ts`:
   - Import `createClient` from `@supabase/supabase-js` using `Deno.env.get("SUPABASE_URL")` and `Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")`.
   - Use the service role client (bypasses RLS) since this is a system-level operation.
   - Call the existing RPC function `get_documents_due_for_reminder()` (created in Plan 01 migration). This returns documents where `next_reminder_date <= NOW()` and `reminder_sent = FALSE`.
   - For each document returned:
     a. Calculate days until expiry: `expiry_date - today`.
     b. Determine reminder type: "30-day" if days > 14, "7-day" if days <= 14 and > 0, "expired" if days <= 0.
     c. INSERT a notification into the notifications table (from Phase 3 COM system) with:
        - `user_id`: the document's `user_id` (landlord)
        - `type`: "compliance_reminder"
        - `title`: "{document_name} {reminder_type} reminder" (e.g., "Gas Safety Certificate expiring in 7 days")
        - `message`: "Your {category} document '{name}' for property {property_id} expires on {expiry_date}. Please renew it promptly."
        - `link`: "/dashboard/landlord/properties/{property_id}/documents"
     d. UPDATE the property_documents record: set `reminder_sent = TRUE`, update `next_reminder_date` to 7 days before expiry (for the 30-day reminder, so the 7-day reminder fires later), or NULL if already past the 7-day mark.
   - Return a JSON response with `{ processed: N, errors: [] }` for observability.
   - Handle errors gracefully: if one document fails, continue processing others. Collect errors in array and return in response.
   - Guard against duplicate notifications: check if a notification with the same document ID and reminder type already exists before inserting.

2. Create pg_cron migration at `britv3.0/supabase/migrations/20260307_epic7_compliance_cron.sql`:
   ```sql
   -- Enable required extensions
   CREATE EXTENSION IF NOT EXISTS pg_cron;
   CREATE EXTENSION IF NOT EXISTS pg_net;

   -- Schedule daily compliance reminder check at 9 AM UTC
   SELECT cron.schedule(
     'compliance-reminders',
     '0 9 * * *',
     $$
     SELECT net.http_post(
       url := current_setting('app.settings.supabase_url') || '/functions/v1/compliance-reminders',
       headers := jsonb_build_object(
         'Authorization', 'Bearer ' || current_setting('app.settings.service_role_key'),
         'Content-Type', 'application/json'
       ),
       body := '{}'::jsonb
     );
     $$
   );
   ```
   Note: The `current_setting('app.settings.supabase_url')` and `current_setting('app.settings.service_role_key')` are Supabase-provided config values. If these are not available in the target environment, use `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from Vault secrets instead. Include a comment in the migration explaining how to adjust for different Supabase project configurations.
