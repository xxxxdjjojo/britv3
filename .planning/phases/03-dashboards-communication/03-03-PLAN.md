  <files>britv3.0/src/services/profile/profile-service.ts, britv3.0/src/app/api/profile/route.ts, britv3.0/src/app/api/profile/picture/route.ts, britv3.0/src/app/api/service-provider/profile/route.ts, britv3.0/src/app/api/notifications/preferences/route.ts</files>
  <action>
**profile-service.ts:**
- `getProfile(userId: string)` -- fetches from profiles table via Supabase server client, returns Profile
- `updateProfile(userId: string, data: ProfileUpdateInput)` -- Zod-validated update of display_name, phone. Sanitize all text inputs via sanitizeText(). Invalidate dashboard cache on update.
- `uploadAvatar(userId: string, file: File)` -- validate file type server-side using `file-type` (magic bytes, must be image/jpeg, image/png, image/webp), resize to 400x400 WebP via `sharp` with quality 85, upload to Supabase Storage bucket "avatars" with path `{userId}/avatar.webp`, set cacheControl '31536000' (1 year), update profiles.avatar_url, return public URL
- `updateProviderProfile(userId: string, data: ProviderProfileInput)` -- upsert to service_provider_details table (or profiles with JSONB column) with services array, coverage_area (postcode list), pricing JSONB. Validate role is 'service_provider'.
- `getNotificationPreferences(userId: string)` -- read from profiles.preferences JSONB
- `updateNotificationPreferences(userId: string, prefs: NotificationPreferences)` -- Zod validate, update profiles.preferences JSONB

**Zod schemas (define in profile-service.ts or separate schema file):**
- `profileUpdateSchema` -- display_name (string, 2-50 chars, trimmed), phone (optional, regex for UK phone)
- `providerProfileSchema` -- services (array of {category, description}), coverage_postcodes (array of strings), pricing (Record<string, number>)
- `notificationPreferencesSchema` -- validate the NotificationPreferences shape

**API route handlers (all use createClient() for server-side auth):**
- `GET /api/profile` -- returns current user's profile
- `PATCH /api/profile` -- updates profile fields
- `POST /api/profile/picture` -- accepts FormData with "file" field, max 5MB
- `GET/PUT /api/service-provider/profile` -- provider extended profile
- `GET/PUT /api/notifications/preferences` -- notification preferences

All routes: check auth via supabase.auth.getUser(), return 401 if not authenticated, return 400 with Zod errors on validation failure, return 500 with generic message on unexpected errors.
