**notification-service.ts:**
- `createPlatformEvent(supabase, event: {event_type, entity_type, entity_id, actor_id, metadata?})` -- single INSERT into platform_events. O(1) write per action. After insert, check if event is critical (new_message, quote_received, booking_confirmed, offer_received) and call email service to send immediate notification. Return the created event.
- `getNotificationFeed(supabase, userId: string, userEntityIds: string[], limit?: number, cursor?: string)` -- query platform_events WHERE entity_id IN (userEntityIds) AND actor_id != userId (don't show own actions). Cursor-based pagination on created_at. LIMIT default 50. JOIN actor display_name.
  - `userEntityIds` is an array of conversation IDs, booking IDs, listing IDs the user is involved in. The caller (API route) assembles this list.
- `getUnreadNotificationCount(supabase, userId: string, userEntityIds: string[], lastReadAt: Date)` -- COUNT of platform_events WHERE entity_id IN (userEntityIds) AND created_at > lastReadAt AND actor_id != userId
- `markAllRead(supabase, userId: string)` -- UPDATE profiles SET notifications_read_at = NOW() WHERE id = userId
- `getUserEntityIds(supabase, userId: string)` -- helper: query conversations, bookings, listings to get all entity IDs the user is involved in. Returns string[].

**email-service.ts:**
- `shouldSendEmail(preferences: NotificationPreferences, eventType: EventType): boolean` -- check per_type[eventType].email, respect quiet_hours
- `sendCriticalEmail(to: string, template: ReactElement, subject: string)` -- rate limit check via @upstash/ratelimit (5 per hour per user email), then Resend send. Return {sent: boolean, rateLimited?: boolean}.
- `sendDailyDigest(to: string, events: PlatformEvent[])` -- render DailyDigest template, send via Resend. Only called from cron route.
- `CRITICAL_EVENTS` -- const set of EventType values that get immediate email: 'quote_received', 'booking_confirmed', 'offer_received'
- `DIGEST_EVENTS` -- all other event types

**Email templates (React Email components):**

- Use @react-email/components (Html, Head, Preview, Body, Container, Heading, Text, Button, Section)

Templates:

- `BookingConfirmed.tsx` -- props: recipientName, serviceName, dateTime, bookingUrl
- `DailyDigest.tsx` -- props: recipientName, events array with title/description/url, date range

**API routes:**
- `GET /api/notifications` -- get notification feed for current user. Calls getUserEntityIds to get relevant entity IDs, then getNotificationFeed. Query params: cursor, limit.
- `GET /api/notifications?count_only=true` -- returns unread count only (uses profile.notifications_read_at)
- `POST /api/notifications/read` -- mark all notifications as read (updates notifications_read_at)
- `POST /api/email/digest` -- cron-triggered route for daily digest. Accepts a secret header (`Authorization: Bearer CRON_SECRET`) to prevent unauthorized calls. For each user with digest_frequency != 'never': gather events since last digest, render DailyDigest template, send via email service. This would be called by Vercel Cron or pg_cron.

**useNotifications.ts** ("use client" hook):
- `useNotifications()` -- useQuery with queryKey ["notifications"], fetches GET /api/notifications, refetchInterval: 60_000 (60s for notifications, less aggressive than inbox)
- `useNotificationCount()` -- useQuery with queryKey ["notification-count"], fetches GET /api/notifications?count_only=true, refetchInterval: 60_000
- `useMarkAllRead()` -- useMutation calling POST /api/notifications/read, on success invalidate ["notification-count"]
