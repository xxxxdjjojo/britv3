**message-service.ts:**
- `getConversations(userId: string, filters?: InboxFilters)` -- query conversations WHERE participant_1_id = userId OR participant_2_id = userId, JOIN to get other participant's display_name, last message preview (content truncated to 100 chars), unread count (messages WHERE created_at > conversation_read_status.last_read_at). Order by last_message_at DESC. Optional filter by context_type.
- `getMessages(conversationId: string, cursor?: string, limit?: number)` -- query messages WHERE conversation_id = conversationId AND (cursor ? created_at < cursor : true), ORDER BY created_at DESC, LIMIT (default 20). JOIN sender display_name.
- `sendMessage(senderId: string, input: SendMessageInput)` -- sanitize content via sanitizeText(), INSERT into messages. UPDATE conversations.last_message_at. If conversation_id not provided, call getOrCreateConversation first. Return the created message.
- `getOrCreateConversation(userId: string, recipientId: string, contextType: ContextType, contextId?: string)` -- check if conversation exists between these two users for this context. If not, INSERT. Return conversation.
- `updateReadStatus(conversationId: string, userId: string)` -- UPSERT conversation_read_status with last_read_at = NOW()
- `getUnreadCount(userId: string)` -- count conversations where last_message_at > last_read_at

Zod schemas: sendMessageSchema (content 1-5000 chars, recipient_id UUID, context_type enum)

**attachment-service.ts:**
- `uploadAttachment(file: File, conversationId: string, userId: string)` -- validate file type via `file-type` magic bytes (allow image/jpeg, image/png, image/webp, application/pdf), reject if >2MB, upload to Supabase Storage "message-attachments" bucket at path `{conversationId}/{messageId}.{ext}`, return URL and type
- `validateAttachment(file: File)` -- returns {valid: boolean, type: 'image' | 'pdf', error?: string}

**API routes:**
- `GET /api/messages` -- list conversations for current user. Query param `count_only=true` returns just unread count.
- `POST /api/messages` -- send new message (creates conversation if needed)
- `GET /api/messages/[conversationId]` -- get messages in thread with cursor pagination via `?cursor=` query param
- `POST /api/messages/[conversationId]` -- send message to existing conversation
- `POST /api/messages/[conversationId]/read` -- update read status
- `POST /api/attachments` -- upload file attachment, return URL

All routes: authenticate via supabase.auth.getUser(), 401 if not auth, validate with Zod.

**useInbox.ts** ("use client" hook):
- `useInbox(filters?)` -- useQuery with queryKey ["inbox", filters], fetches GET /api/messages, refetchInterval: 30_000, refetchIntervalInBackground: false, staleTime: 10_000
- `useUnreadCount()` -- useQuery with queryKey ["unread-count"], fetches GET /api/messages?count_only=true, refetchInterval: 30_000

**useMessages.ts** ("use client" hook):
- `useMessages(conversationId)` -- useInfiniteQuery for cursor-based pagination. queryKey ["messages", conversationId], getNextPageParam extracts cursor from last message's created_at. Fetches GET /api/messages/[conversationId]?cursor=X
- `useSendMessage()` -- useMutation calling POST /api/messages/[conversationId]. On success: invalidate ["messages", conversationId] and ["inbox"] queries, show sonner toast
- `useMarkAsRead(conversationId)` -- useMutation calling POST /api/messages/[conversationId]/read. Fire on mount when opening a conversation.

**InboxList.tsx** ("use client"):

**MessageThread.tsx** ("use client"):
