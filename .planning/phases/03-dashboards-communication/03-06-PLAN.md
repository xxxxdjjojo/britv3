  <action>
**dashboard-service.ts:**
- `getDashboardData(userId: string, role: UserRole)` -- check Redis cache first with key `dashboard:${userId}`. If cached, return {data, cached: true}. If not, build aggregated query based on role:
  - For each role, query relevant tables and aggregate into the role-specific dashboard shape
  - homebuyer: count saved_properties, count active searches, list upcoming viewings, recent activity
  - renter: count saved_rentals, list application statuses, tenancy details
  - seller: list user's listings with views/saves/enquiries counts, viewing requests, offers received
  - landlord: portfolio property count, occupancy rate, total income, properties summary
  - agent: active listings count, leads by stage, upcoming viewings, revenue total
  - provider: verification status, active jobs count, avg rating, total earnings, pending quotes
  - NOTE: Many of these will return placeholder/stub data since the underlying tables (listings, bookings, etc.) may not exist yet. The service should handle missing tables gracefully with try/catch and return zero/empty defaults.
  - Cache result in Redis with 300s (5-minute) TTL
  - Return {data, cached: false}

- `invalidateDashboardCache(userId: string)` -- delete Redis key `dashboard:${userId}`

- `getActivityLog(userId: string, cursor?: string, limit?: number)` -- query activity_log WHERE user_id = userId, cursor-based pagination on created_at. Default limit 20. Return {entries: ActivityLogEntry[], nextCursor: string | null}.

- `logActivity(userId: string, eventType: string, description: string, metadata?: Record<string, unknown>)` -- INSERT into activity_log. Used by other services to record user actions.

**API route GET /api/dashboard:**
- Authenticate user
- Get user's active_role from profile
- Call getDashboardData(userId, role)
- Return JSON with data + cached flag
- Query param `?refresh=true` bypasses cache (invalidates first)
- Query param `?activity=true&cursor=X` returns activity log instead of dashboard data

**useDashboard.ts** ("use client" hook):
- `useDashboard()` -- useQuery with queryKey ["dashboard"], fetches GET /api/dashboard, staleTime: 300_000 (5 min, matches Redis TTL)
- `useRefreshDashboard()` -- function that invalidates query cache and refetches with ?refresh=true
- `useActivityLog()` -- useInfiniteQuery with queryKey ["activity-log"], fetches GET /api/dashboard?activity=true&cursor=X

**useRealtime.ts** ("use client" hook):
- `useRealtimeSubscription(userId: string, onUpdate: () => void)` -- subscribes to Supabase Realtime channel `dashboard:${userId}`. Listens for postgres_changes on relevant tables (conversations, platform_events). On change, calls onUpdate (which should invalidate dashboard query).
- Uses a single channel per user (not per table) to minimize connection count.
- Cleanup on unmount via useEffect return.

**DashboardShell.tsx** (server component wrapper, with client children):
- Props: role (UserRole), title, children
