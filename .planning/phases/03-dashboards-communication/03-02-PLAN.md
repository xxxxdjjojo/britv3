**redis.ts:**
Create Upstash Redis client singleton with helper functions:
- `redis` -- Redis instance from `@upstash/redis` using env vars `UPSTASH_REDIS_URL` and `UPSTASH_REDIS_TOKEN`
- `getCached<T>(key: string)` -- returns cached value or null
- `setCache(key: string, value: unknown, ttlSeconds: number)` -- sets with TTL via `setex`
- `invalidateCache(key: string)` -- deletes key
- `invalidateCachePattern(pattern: string)` -- scan + delete matching keys (for `dashboard:*` invalidation)
- Rate limiter factory using `@upstash/ratelimit` Ratelimit.slidingWindow for email rate limiting (5 per hour)

**sanitize.ts:**
- `sanitizeHtml(dirty: string): string` -- uses isomorphic-dompurify with ALLOWED_TAGS for basic formatting (b, i, a, p, br, ul, ol, li)
- `sanitizeText(dirty: string): string` -- strips ALL HTML tags, returns plain text only
- Both functions handle null/undefined gracefully (return empty string)

**compress-image.ts:**
Client-side image compression utility (from research):
- `compressImage(file: File, maxSizeKB?: number, maxWidth?: number): Promise<Blob>` -- defaults 500KB, 1200px width
- Uses Canvas API, outputs JPEG at quality 0.8
- Strips EXIF by redrawing on canvas

**QueryProvider.tsx:**
- `"use client"` component wrapping QueryClientProvider
- QueryClient with defaultOptions: staleTime 60_000, retry 1
- Uses useState pattern to avoid re-creating client on re-renders (from research)
