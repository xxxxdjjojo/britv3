**milestone-service.ts:**
- `initializeTransactionMilestones(supabase, transactionId: string)` -- INSERT 8 rows into transaction_milestones using TRANSACTION_MILESTONE_TEMPLATE, all with status 'not_started'. Called when a new transaction is created.
- `initializeJobMilestones(supabase, bookingId: string)` -- INSERT 5 rows into service_job_milestones using SERVICE_JOB_MILESTONE_TEMPLATE.
- `getTransactionMilestones(supabase, transactionId: string)` -- SELECT all milestones for transaction, ORDER BY the template order
- `updateTransactionMilestone(supabase, milestoneId: string, userId: string, data: {status: MilestoneStatus, notes?: string, completed_date?: string})` -- UPDATE the milestone. If status changed to 'completed', set completed_date. If status changed to 'in_progress', clear completed_date. Create a platform_event for milestone_updated.
- `getJobMilestones(supabase, bookingId: string)` -- same pattern as transaction
- `updateJobMilestone(supabase, milestoneId: string, userId: string, data: {...})` -- same as transaction update
- `getTransactionProgress(supabase, transactionId: string)` -- returns {completed: number, total: 8, percentage: number}
- `getJobProgress(supabase, bookingId: string)` -- returns {completed: number, total: 5, percentage: number}

Zod schemas:
- `updateMilestoneSchema` -- status (enum), notes (optional string max 500 chars), completed_date (optional ISO date string)

**API routes:**
- `GET /api/milestones/transaction?id={transactionId}` -- returns milestone list with progress
- `PATCH /api/milestones/transaction?id={milestoneId}` -- update milestone status/notes
- `POST /api/milestones/transaction` -- initialize milestones for new transaction (body: {transaction_id})
- Same pattern for `/api/milestones/job` with booking_id

All routes authenticate and validate.

**MilestoneTracker.tsx:**
- Generic progress stepper component
- Props: milestones (array of {key, label, description, status, notes, completedDate}), onUpdate (callback), readOnly
- Vertical stepper with connected line between steps
- Each step: circle icon (CheckCircle for completed green, Circle for in_progress amber, CircleDot for not_started grey), label, description, status badge
- Completed steps show completed_date
- In-progress step is highlighted/expanded
- If not readOnly: click step to expand dialog/popover for updating status (Select dropdown) and adding notes (Textarea)
- Progress bar at top showing X/Y completed (uses Shadcn Progress component)
- Use Shadcn: Card, Progress, Select, Textarea, Button, Badge, Tooltip

**TransactionMilestones.tsx** ("use client"):
- Props: transactionId
- Fetches milestones via GET /api/milestones/transaction?id=X
- Renders MilestoneTracker with the 8 UK property pipeline steps
- Labels: "Offer Accepted", "Mortgage Submitted", "Survey Instructed", "Survey Completed", "Conveyancing Started", "Searches Completed", "Contracts Exchanged", "Completion"
- On update: PATCH /api/milestones/transaction?id=milestoneId, refetch, show sonner toast

**JobMilestones.tsx** ("use client"):
- Props: bookingId
- Same pattern but for 5 service job steps
- Labels: "Quote Accepted", "Job Scheduled", "Work Started", "Work Completed", "Payment Received"

**FilesTab.tsx** ("use client"):
- Props: contextType (ContextType), contextId (string)
- Queries messages with attachments for the given context: GET /api/messages for the conversation matching context_type + context_id, filter where attachment_url is not null
- Renders as a Table (Shadcn Table): columns for file name, type (image/pdf), size, uploaded by, date
- Click row opens file in new tab
- Upload button for direct file upload to the conversation
- Empty state: "No files shared yet"
- Groups by type (images, documents)

**Pages:**
- `milestones/transaction/[id]/page.tsx` -- server component, renders TransactionMilestones with params.id
- `milestones/job/[bookingId]/page.tsx` -- server component, renders JobMilestones with params.bookingId
