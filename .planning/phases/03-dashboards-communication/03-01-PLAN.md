<task type="auto">
  <name>Task 3: Create database migration for Phase 3 tables</name>
  <files>britv3.0/supabase/migrations/003_dashboards_communication.sql</files>
  <action>
Create SQL migration file with all Phase 3 tables. Follow the schema from the research document exactly.

**Tables to create:**

1. `conversations` -- participant_1_id, participant_2_id, context_type (CHECK IN listing/booking/rfq/general), context_id, last_message_at. FK to auth.users. Indexes on both participants + last_message_at DESC.

2. `messages` -- conversation_id (FK CASCADE), sender_id (FK auth.users), content (CHECK LENGTH <= 5000), attachment_url, attachment_type (CHECK IN image/pdf), attachment_size_bytes. Index on (conversation_id, created_at DESC).

3. `conversation_read_status` -- composite PK (conversation_id, user_id), last_read_at. FK to conversations CASCADE, FK to auth.users.

4. `platform_events` -- BIGSERIAL PK, event_type, entity_type (CHECK IN conversation/booking/listing/rfq/transaction), entity_id, actor_id (FK auth.users), metadata JSONB default '{}'. Index on (entity_id, created_at DESC). Partial index for cleanup (created_at < NOW() - 90 days).

5. `market_pricing` -- service_category, region, price_low/median/high NUMERIC(10,2), sample_size, data_source (CHECK IN seed/platform/blended), last_updated. UNIQUE(service_category, region).

6. `transaction_milestones` -- transaction_id UUID (NO FK yet -- transactions table comes in Phase 6), milestone_key (CHECK IN 8 steps), status default 'not_started', updated_by (FK auth.users), updated_at, notes, completed_date DATE. UNIQUE(transaction_id, milestone_key). Index on transaction_id.

7. `service_job_milestones` -- booking_id UUID (NO FK yet -- bookings table comes in Phase 4), milestone_key (CHECK IN 5 steps), status default 'not_started', updated_by (FK auth.users), updated_at, notes. UNIQUE(booking_id, milestone_key). Index on booking_id.

8. `activity_log` -- PARTITION BY RANGE (created_at). Columns: id BIGSERIAL, user_id (FK auth.users), event_type TEXT, description TEXT, metadata JSONB, created_at TIMESTAMPTZ default NOW(). Create initial partitions for next 12 months using native PostgreSQL partitioning (not pg_partman). Index on (user_id, created_at DESC).

**RLS on ALL tables:**
- `conversations`: SELECT/INSERT where participant_1_id or participant_2_id = auth.uid()
- `messages`: SELECT where conversation_id in user's conversations; INSERT where sender_id = auth.uid() AND conversation_id in user's conversations
- `conversation_read_status`: ALL where user_id = auth.uid()
- `platform_events`: SELECT where actor_id = auth.uid() OR entity_id in user's relevant entities; INSERT where actor_id = auth.uid()
- `market_pricing`: SELECT for all authenticated/anon; no INSERT/UPDATE/DELETE for regular users
- `transaction_milestones`: SELECT/UPDATE where updated_by = auth.uid() (simplified -- real auth comes later)
- `service_job_milestones`: same pattern as transaction_milestones
- `activity_log`: SELECT where user_id = auth.uid(); INSERT where user_id = auth.uid()

Also add to profiles table:
- `preferences` JSONB column DEFAULT '{}' (for notification preferences)
- `notifications_read_at` TIMESTAMPTZ (for tracking last-seen notifications)

Comment the SQL thoroughly explaining each table's purpose and which requirement it serves.
