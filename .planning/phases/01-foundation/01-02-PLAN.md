Create `britv3.0/supabase/migrations/001_foundation.sql` with the complete Phase 1 schema. Follow the schema from 01-RESEARCH.md closely but ensure completeness:

**Extensions:**
- `uuid-ossp`

**Enum types:**
- `user_role`: homebuyer, renter, seller, landlord, agent, service_provider
- `verification_level`: basic, standard, enhanced, professional
- `verification_stage`: email, phone, identity, insurance, qualifications, admin_review

**Tables:**
1. `profiles` -- extends auth.users via FK: id (UUID PK ref auth.users ON DELETE CASCADE), display_name, active_role (user_role default 'homebuyer'), verification_level (default 'basic'), avatar_url, phone, phone_verified (default false), is_admin (default false), created_at, updated_at, deleted_at (nullable, for soft delete)

2. `user_roles` -- junction table for multi-role: id (UUID PK), user_id (ref auth.users ON DELETE CASCADE), role (user_role), granted_at. UNIQUE(user_id, role)

3. `provider_verifications` -- verification pipeline: id (UUID PK), user_id (ref auth.users ON DELETE CASCADE), stage (verification_stage), status (text CHECK pending/submitted/approved/rejected), document_url, reviewed_by (ref auth.users nullable), reviewed_at, rejection_reason, created_at, updated_at. UNIQUE(user_id, stage)

4. `consent_records` -- GDPR consent: id (UUID PK), user_id (ref auth.users ON DELETE CASCADE), consent_type (text CHECK marketing/analytics/third_party), granted (boolean default false), ip_address (inet), user_agent (text), created_at, updated_at. UNIQUE(user_id, consent_type)

5. `consent_audit_log` -- immutable audit trail: id (bigserial PK), user_id (UUID NOT NULL), consent_type, old_value (boolean nullable), new_value (boolean), ip_address, user_agent, created_at

6. `auth_audit_log` -- auth event tracking: id (bigserial PK), user_id (ref auth.users ON DELETE SET NULL), event_type, event_details (jsonb), ip_address, user_agent, created_at

7. `deletion_requests` -- GDPR Art. 17: id (UUID PK), user_id (ref auth.users), requested_at, scheduled_purge_at (NOW() + 30 days), purged_at, status (CHECK pending/processing/completed/cancelled)

**Indexes:** As specified in 01-RESEARCH.md -- on active_role (partial WHERE deleted_at IS NULL), user_roles(user_id), user_roles(role), consent_records(user_id), consent_audit_log(user_id, created_at DESC), auth_audit_log(user_id, created_at DESC), provider_verifications(user_id), provider_verifications(status WHERE pending)

**Triggers:**
1. `update_updated_at()` function + triggers on profiles, consent_records, provider_verifications
2. `handle_new_user()` -- AFTER INSERT ON auth.users: creates profile row + auth_audit_log entry. Uses SECURITY DEFINER.
3. `log_consent_change()` -- AFTER INSERT OR UPDATE ON consent_records: inserts into consent_audit_log. Uses SECURITY DEFINER.

**RLS Policies:** Enable RLS on ALL 7 tables. Create policies:
- profiles: users read/update own (WHERE deleted_at IS NULL for SELECT)
- user_roles: users read own roles
- consent_records: users read/update/insert own consent
- consent_audit_log: users read own audit trail
- auth_audit_log: users read own events
- provider_verifications: users read/insert own verifications
- deletion_requests: users read/insert own requests
