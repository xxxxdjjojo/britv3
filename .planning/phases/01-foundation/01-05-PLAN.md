    - Unauthenticated request to /dashboard redirects to /login

  <action>
1. Write tests first:
   - `csp.test.ts`: Verify CSP header format, nonce presence, required domains in allowlist, security headers
   - `middleware.test.ts`: Verify redirect logic for unauthenticated to protected routes, authenticated to auth routes, public route passthrough

2. Create `src/middleware.ts` following the pattern from 01-RESEARCH.md:
   - Create Supabase server client using `createServerClient` from `@supabase/ssr` (NOT the helper from lib/supabase -- middleware needs direct cookie access via request.cookies)
   - Call `supabase.auth.getUser()` (NOT getSession) to verify auth
   - Route protection logic:
     * PUBLIC_ROUTES (/, /about, /terms, /privacy) -- always accessible
     * AUTH_ROUTES (/login, /register, /forgot-password, /reset-password, /verify-email) -- redirect to /dashboard if authenticated
     * Everything else -- redirect to /login if NOT authenticated
   - CSP Level 3 with nonce:
     * Generate nonce via `crypto.randomUUID()` base64 encoded
     * `default-src 'self'`
     * `script-src 'self' 'nonce-{nonce}' https://accounts.google.com https://appleid.cdn-apple.com`
     * `style-src 'self' 'unsafe-inline'` (Tailwind needs unsafe-inline)
     * `img-src 'self' data: blob: https://*.supabase.co`
     * `connect-src 'self' https://*.supabase.co wss://*.supabase.co`
     * `frame-src https://accounts.google.com https://appleid.apple.com`
     * `form-action 'self'`
     * `base-uri 'self'`
     * `frame-ancestors 'none'`
   - Set `x-nonce` response header for server components to read
   - Additional security headers: X-Frame-Options DENY, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin, Permissions-Policy camera=() microphone=() geolocation=()
   - Matcher config: exclude `_next/static`, `_next/image`, `favicon.ico`, static file extensions
  </action>
