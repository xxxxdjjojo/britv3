   - `getVerificationStatus(userId: string)` -- select from provider_verifications WHERE user_id, ordered by stage
   - `getVerificationProgress(userId: string)` -- fetches status, computes:
     * completedStages: stages with status='approved'
     * currentStage: first non-approved stage
     * level: computeVerificationLevel(completedStages)
     * percentage: (approved count / total stages) * 100
   - `submitVerification(userId: string, stage: VerificationStage, documentUrl?: string)` -- validates stage order (all previous stages must be approved or current), inserts/upserts with status='submitted'
   - Stage order: email -> phone -> identity -> insurance -> qualifications -> admin_review
   - Email stage is auto-approved if user's email is verified in auth.users

   - Active stage shows action: "Verify Phone" button, "Upload ID" file input, etc.
