  <behavior>
    - signUp(email, password, displayName) calls supabase.auth.signUp with emailRedirectTo, returns { user, error }
    - signIn(email, password) calls supabase.auth.signInWithPassword, returns { user, session, error }
    - signInWithOAuth(provider) calls supabase.auth.signInWithOAuth with PKCE options, redirectTo callback URL
    - signOut() calls supabase.auth.signOut
    - resetPassword(email) calls supabase.auth.resetPasswordForEmail with redirect URL
    - updatePassword(newPassword) calls supabase.auth.updateUser
    - useAuth hook provides { user, loading, signIn, signUp, signOut, signInWithOAuth } state
    - PKCE callback route exchanges code for session and redirects
  </behavior>
  <action>
1. Write tests first using the Supabase mock from `__tests__/mocks/supabase.ts`:
   - `signup.test.ts`: Test signUp calls supabase correctly, handles errors (weak password, duplicate email)
   - `oauth.test.ts`: Test signInWithOAuth calls with correct provider ('google', 'apple'), PKCE options
   - `password-reset.test.ts`: Test resetPassword sends email, updatePassword changes password
   - `session.test.ts`: Test useAuth hook initializes with loading state, resolves user
   - `verify-email.test.ts`: Test callback route handles code exchange success and error

2. Create `src/services/auth/auth-service.ts`:
   - `signUp(email: string, password: string, displayName: string)` -- calls `supabase.auth.signUp` with `data: { display_name: displayName }` in options and `emailRedirectTo` pointing to `/auth/callback`
   - `signIn(email: string, password: string)` -- calls `supabase.auth.signInWithPassword`
   - `signInWithOAuth(provider: 'google' | 'apple')` -- calls `supabase.auth.signInWithOAuth` with `redirectTo: origin + '/auth/callback'`, `queryParams: { access_type: 'offline', prompt: 'consent' }` for Google
   - `signOut()` -- calls `supabase.auth.signOut`
   - `resetPassword(email: string)` -- calls `supabase.auth.resetPasswordForEmail` with `redirectTo: origin + '/reset-password'`
   - `updatePassword(password: string)` -- calls `supabase.auth.updateUser({ password })`
   - `getUser()` -- calls `supabase.auth.getUser()` (NOT getSession)

3. Create `src/hooks/useAuth.ts`:
   - Uses `useState` for `user` and `loading`
   - On mount, calls `supabase.auth.getUser()` to initialize
   - Subscribes to `supabase.auth.onAuthStateChange` for real-time updates
   - Exposes auth operations from auth-service
   - Cleans up subscription on unmount

4. Create `src/app/auth/callback/route.ts`:
   - GET handler that reads `code` from searchParams
   - Calls `supabase.auth.exchangeCodeForSession(code)` via server client
   - On success: redirect to `/dashboard` (or `next` searchParam)
   - On error: redirect to `/login?error=auth_callback_error`
  </action>
