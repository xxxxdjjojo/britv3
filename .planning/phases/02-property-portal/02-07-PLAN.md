    **`britv3.0/src/components/listings/ListingForm.tsx`** ("use client"):
    Multi-step form with 5 steps using react-hook-form:
    - Step indicator at top (1-5 numbered steps with labels, current step highlighted)
    - Back/Next navigation buttons, "Publish" on final step
    - Form state persists across steps (single react-hook-form instance)
    - Zod validation per step (only validate current step's fields on Next)
    - Props: `mode: "create" | "edit"`, `initialData?: ListingWithProperty` for edit mode
    - On submit: POST to /api/listings (create) or PATCH to /api/listings/[id] (edit)
    - After successful create: redirect to listing detail/edit page
    - After media upload step: upload images via /api/listings/[id]/media

    **Step 1: PropertyDetails.tsx:**
    - Listing type: Sale / Rent radio buttons
    - Address: line1, line2, city, county, postcode (postcode with autocomplete via usePostcodeAutocomplete)
    - Property type: select dropdown (all PropertyType values)
    - Bedrooms: number input (0-50)
    - Bathrooms: number input (0-20, allows 0.5 increments)
    - Reception rooms: number input (0-10)
    - Square footage: number input (optional)
    - Year built: number input (optional, 1600-2050)
    - Tenure: select (freehold, leasehold, shared_ownership)
    - New build: checkbox
    - Validation: address, property_type, bedrooms, bathrooms required

    **Step 2: Description.tsx:**
    - Title: text input (max 200 chars, character counter)
    - Description: textarea (max 5000 chars, character counter)
    - Features: tag input -- user types features and presses Enter to add as tags (stored as JSONB array)
    - EPC rating: select A-G (optional)
    - EPC score: number input 1-100 (optional, only if rating provided)
    - Council tax band: select A-H (optional)

    **Step 3: Pricing.tsx:**
    - Price: number input with GBP prefix (required)
    - Price qualifier: select (Offers Over, Guide Price, Fixed Price, From, POA)
    - For rent listings: Rent frequency (weekly, monthly, yearly)
    - Service charge annual: number input (optional, for leasehold)
    - Ground rent annual: number input (optional, for leasehold)
    - Available from: date picker (optional)

    **Step 4: MediaUpload.tsx:**
    - Uses ImageUploader component for photos
    - Separate section for floor plan upload (accepts images + PDF)
    - EPC document upload (accepts images + PDF)
    - Shows upload progress and thumbnails
    - Drag to reorder photos (or simple up/down buttons)
    - Note: Images upload immediately to /api/listings/[id]/media on drop/select (not on final submit). Requires listing to be created first as draft -- so the form creates a draft listing on Step 1 completion, then uses that ID for media uploads.

    **Step 5: Review.tsx:**
    - Summary view of all entered data
    - Thumbnail grid of uploaded photos
    - "Edit" links next to each section to jump back to that step
    - "Save as Draft" and "Publish" buttons
    - Publish changes listing status from 'draft' to 'active' via PATCH

    **`britv3.0/src/components/listings/ImageUploader.tsx`** ("use client"):
    - Uses `react-dropzone` for drag-and-drop area
    - Accepts JPEG, PNG, WebP (validates client-side)
    - On file drop/select: compress via `compressMultiple()` from lib/upload/compress.ts
    - Show individual progress bars during compression + upload
    - Display thumbnails in a reorderable grid after upload
    - Delete button on each thumbnail
    - Max 30 images indicator ("5 of 30 photos")
    - Uses useImageUpload hook for upload state management

    **`britv3.0/src/hooks/useImageUpload.ts`** ("use client"):
    - Manages upload state: uploading files, progress per file, completed uploads, errors
    - `uploadImages(files: File[], listingId: string)`: compress then POST each to /api/listings/[id]/media
    - `deleteImage(mediaId: string, listingId: string)`: DELETE to media route
    - `reorderImages(mediaIds: string[], listingId: string)`: PATCH to media route
    - Uses react-query mutations for each operation

    **Page routes:**
    - `listings/new/page.tsx`: Renders ListingForm with mode="create". Requires auth + role check (agent or seller).
    - `listings/[id]/page.tsx`: Fetches listing, renders ListingForm with mode="edit" and initialData. Verifies ownership.

    Use Shadcn components throughout: Input, Textarea, Select, Button, Card, Badge, Progress, Label, Tabs. Follow react-hook-form + zodResolver pattern from Phase 1 auth forms.
