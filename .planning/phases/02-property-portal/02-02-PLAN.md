    - searchProperties({ sort: "price_asc" }) orders by price ascending
    - searchProperties({ sort: "relevance", q: "garden" }) orders by ts_rank
    - searchProperties({ cursor: "uuid-value", per_page: 20 }) returns next page
    - searchProperties returns { data, count, cursor }

    **`britv3.0/src/services/search/geocode-service.ts`:**
    - Pure async functions wrapping postcodes.io REST API (follow research Pattern 5)
    - `geocodePostcode(postcode: string): Promise<GeocodedLocation | null>`
    - `autocompletePostcode(partial: string): Promise<string[]>`
    - `reverseGeocode(lat: number, lng: number): Promise<GeocodedLocation | null>`

    **`britv3.0/src/lib/search/query-builder.ts`:**
    - `buildSearchQuery(supabase, params: SearchParams)` function that constructs the Supabase query
    - If lat/lng present: use `supabase.rpc("search_listings_by_radius", ...)` with radius conversion (miles to meters * 1609.34)
    - Otherwise: query `search_listings` materialized view with chained filters
    - Apply filters: listing_type, min_price (.gte), max_price (.lte), min_bedrooms (.gte), property_type (.in), epc_rating (.lte), new_build (.eq), amenities (JSONB containment)
    - Apply full-text search: if `q` param, use `.textSearch("description_tsv", q)`
    - Apply sort: price_asc/desc (.order price), date_desc/asc (.order listed_date), relevance (ts_rank via textSearch)
    - Apply cursor pagination: if cursor, use `.gt("listing_id", cursor)` and `.limit(per_page)`
    - Return { data, count, cursor: last item's listing_id or null }

    - `searchProperties(params: SearchParams): Promise<SearchResult>`
    - Creates Supabase server client, delegates to query-builder
    - Logs to search_analytics table (fire-and-forget, don't await)
    - Follow research Pattern 1 exactly
