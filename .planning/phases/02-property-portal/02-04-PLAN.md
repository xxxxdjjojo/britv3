    - `propertiesToGeoJSON(properties: PropertyMapPoint[]): GeoJSON.FeatureCollection` -- converts property array to GeoJSON FeatureCollection with Point geometries. Coordinates in [lng, lat] order (GeoJSON convention, NOT lat/lng).
    - Cluster configuration constants: `CLUSTER_MAX_ZOOM = 14`, `CLUSTER_RADIUS = 60`

    **`britv3.0/src/components/map/PropertyMap.tsx`:**
    "use client" component. Follow research Pattern 3 closely:
    - Import `Map, Source, Layer` from `@vis.gl/react-maplibre`
    - Import `maplibre-gl/dist/maplibre-gl.css`
    - Props: `properties: PropertyMapPoint[]`, `onPropertyClick?: (id: string) => void`, `onBoundsChange?: (bounds: MapBounds) => void`, `initialViewState?: MapViewState`, `children?: ReactNode` (for draw tool)
    - Default initialViewState centers on London: lng -0.1276, lat 51.5074, zoom 10
    - MapTiler style URL from `process.env.NEXT_PUBLIC_MAPTILER_API_KEY`
    - Use MapLibre native GeoJSON clustering (via Source `cluster={true}` props, NOT supercluster) -- this is simpler and sufficient for MVP
    - Three Layer components:
      1. `clusters` layer: circle type, filter `["has", "point_count"]`, Britestate green (#1B4D3E), radius scales by point_count via `["step", ["get", "point_count"], 20, 10, 30, 50, 40]`
      2. `cluster-count` layer: symbol type, shows `{point_count_abbreviated}` in white
      3. `unclustered-point` layer: circle type, filter `["!", ["has", "point_count"]]`, smaller radius (8), Britestate green
    - On cluster click: get cluster expansion zoom via `getClusterExpansionZoom`, fly to cluster coordinates
    - On unclustered point click: call `onPropertyClick` with feature property `id`
    - On `moveend` event: extract bounds (getNorth/getSouth/getEast/getWest), call `onBoundsChange`
    - Set map container height to 100% (parent must set height)
