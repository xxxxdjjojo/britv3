    **Step 2: Create migration file** `britv3.0/supabase/migrations/002_property_portal.sql`:

    Follow the exact schema from the research document (Pattern: Database Schema section). Include:

    1. **Extensions**: `CREATE EXTENSION IF NOT EXISTS postgis;` and `CREATE EXTENSION IF NOT EXISTS pg_trgm;`

    2. **Enums**: `property_type` (detached, semi_detached, terraced, flat, bungalow, land, cottage, penthouse, studio, maisonette, other), `listing_type` (sale, rent), `listing_status` (draft, active, under_offer, sold, let, withdrawn, archived), `tenure_type` (freehold, leasehold, shared_ownership), `media_type` (image, floor_plan, epc_document)

    3. **Tables** (use exact column definitions from research):
       - `properties` -- physical asset with PostGIS GEOGRAPHY(POINT, 4326), tsvector for FTS, JSONB features, postcode regex constraint
       - `listings` -- market offering with price, status, listing_type, slug, analytics counters
       - `property_media` -- images/floor plans with sort_order, thumbnail_url
       - `price_history` -- triggered on listing price changes
       - `saved_properties` -- user shortlist with UNIQUE(user_id, listing_id)
       - `saved_searches` -- JSONB filters, alert preferences
       - `search_analytics` -- query logging for insights
       - `viewing_history` -- track which listings user has viewed

    4. **Triggers**:
       - `update_properties_tsv` trigger on properties INSERT/UPDATE to auto-generate `description_tsv` from `title` (weight A) + `description` (weight B) + `city` (weight C)
       - `update_properties_updated_at` trigger
       - `update_listings_updated_at` trigger
       - `track_price_changes` trigger on listings UPDATE when `OLD.price != NEW.price`, inserts into price_history
       - `generate_listing_slug` trigger on listings INSERT to generate slug from property address + listing_type

    5. **Indexes** (critical for search performance):
       - GIST index on `properties.coordinates`
       - GIN index on `properties.description_tsv`
       - GIN index on `properties.features`
       - Composite index on `listings(listing_type, status, price)` for filtered searches
       - Index on `listings(listed_date DESC)` for sorting
       - Index on `listings(user_id)` for my-listings queries
       - Index on `property_media(listing_id, sort_order)`
       - Unique index on `saved_properties(user_id, listing_id)` -- already from UNIQUE constraint
       - Index on `saved_searches(user_id)`

    6. **Materialized View** `search_listings`:
       - Joins properties + listings + primary image (sort_order = 0)
       - Includes all searchable columns: listing_id, property_id, listing_type, status, price, property_type, bedrooms, bathrooms, city, postcode, coordinates, description_tsv, features, epc_rating, new_build, listed_date, slug, thumbnail_url, title, address_line1
       - Filter: `listings.status = 'active' AND listings.deleted_at IS NULL AND properties.deleted_at IS NULL`
       - Create UNIQUE INDEX on `search_listings(listing_id)` for CONCURRENTLY refresh

    7. **RPC Functions**:
       - `search_listings_by_radius(center_lat, center_lng, radius_meters, filters...)` -- PostGIS radius search
       - `search_listings_by_polygon(polygon_geojson TEXT, filters...)` -- PostGIS polygon search using ST_Within with bounding box pre-filter
       - `refresh_search_listings()` -- wrapper for REFRESH MATERIALIZED VIEW CONCURRENTLY

    8. **RLS Policies**:
       - `properties`: SELECT for all authenticated, INSERT/UPDATE for owner (via listing user_id), DELETE disabled
       - `listings`: SELECT for active listings (anyone), SELECT all for owner, INSERT for authenticated, UPDATE/DELETE for owner
       - `property_media`: SELECT via listing access, INSERT/UPDATE/DELETE for listing owner
       - `saved_properties`: Full CRUD for own records only (user_id = auth.uid())
       - `saved_searches`: Full CRUD for own records only
       - `price_history`: SELECT for all authenticated, INSERT restricted to trigger
       - `search_analytics`: INSERT for authenticated, SELECT for admin only
       - `viewing_history`: Full CRUD for own records

    9. **Storage Buckets** (via SQL or comment for manual creation):
       - `property-images` bucket: public read, authenticated write with folder path validation
       - `property-documents` bucket: private, authenticated read/write for owners
