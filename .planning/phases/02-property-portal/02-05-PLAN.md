    - updateAlertPreferences(userId, searchId, prefs) updates alert settings

  </behavior>
  <action>
    **`britv3.0/src/services/saved/saved-searches-service.ts`:**
    Pure async functions:
    - `saveSearch(userId: string, name: string, filters: SearchFilters, alertPrefs: { enabled: boolean; frequency: 'instant' | 'daily' | 'weekly' })`: Insert into saved_searches with filters as JSONB
    - `deleteSearch(userId: string, searchId: string)`: Delete where id AND user_id
    - `getSavedSearches(userId: string)`: Query saved_searches for user, ordered by created_at DESC
    - `updateAlertPreferences(userId: string, searchId: string, prefs: { alerts_enabled: boolean; alert_frequency: string })`: Update alert columns
    - `checkNewResults(searchId: string)`:
      1. Load saved search record
      2. Parse filters from JSONB
      3. Call searchProperties with those filters + `listed_after: last_alerted_at` (add this filter support to search service)
      4. Update new_results_count on the saved search
      5. Return count of new results
    - `markAlerted(searchId: string)`: Update last_alerted_at to NOW()

    **`britv3.0/src/app/api/saved/searches/route.ts`:**
    - GET: returns `getSavedSearches(userId)` -- requires auth
    - POST: body `{ name, filters, alerts_enabled?, alert_frequency? }`, validates filters against SearchFilters schema, calls `saveSearch`, returns 201
    - DELETE: body `{ search_id }`, calls `deleteSearch`, returns 204
    - PATCH: body `{ search_id, alerts_enabled?, alert_frequency? }`, calls `updateAlertPreferences`, returns 200

    **`britv3.0/src/hooks/useSavedSearches.ts`:**
    "use client" hook:
    - `useSavedSearches()` returns { searches, isLoading }
    - `useSaveSearch()` returns mutation to save current search filters
    - `useDeleteSearch()` returns mutation
    - `useLoadSearch()` returns callback that sets URL params from saved filters (via nuqs)
    - Query key: `["saved-searches"]`
