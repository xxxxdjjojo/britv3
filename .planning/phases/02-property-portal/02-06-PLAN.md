    **`britv3.0/src/app/(main)/search/page.tsx`** (Server Component):
    - Follow research TanStack Query prefetch pattern
    - Extract search params from URL (Next.js searchParams prop)
    - Create QueryClient, prefetch search results with `queryClient.prefetchQuery`
    - Wrap SearchPage in `HydrationBoundary` with dehydrated state
    - Set metadata: title "Search Properties | Britestate", description based on filters

    **`britv3.0/src/components/search/SearchPage.tsx`** ("use client"):
    - Orchestrator component that composes all search UI
    - Layout: SearchBar at top, then a row with SearchFilters (left sidebar, collapsible on mobile) + content area
    - Content area has SearchSortBar + view toggle (list/map/split) + SearchResults or PropertyMap or split view
    - View modes: "list" (default), "map" (full map), "split" (50/50 list+map on desktop)
    - Uses useSearchResults() hook for data
    - Passes results to both SearchResults (list) and PropertyMap (map)
    - Shows result count ("X properties found")
    - Infinite scroll: load more when user scrolls near bottom (or "Load More" button)

    **`britv3.0/src/components/search/SearchBar.tsx`** ("use client"):
    - Location input with postcode autocomplete dropdown
    - Uses usePostcodeAutocomplete for suggestions (debounced 300ms)
    - On postcode selection: geocode it, update search params with lat/lng
    - Also accepts area names (not just postcodes)
    - Sale/Rent toggle (listing_type)
    - Search button triggers filter application
    - Styled with Shadcn Input, Button, and a custom autocomplete dropdown

    **`britv3.0/src/components/search/SearchFilters.tsx`** ("use client"):
    - Filter panel with:
      - Property type: multi-select checkboxes (detached, semi, terraced, flat, etc.)
      - Price range: min/max inputs (preset ranges for sale: 100K-2M+, for rent: 500-5000+/mo)
      - Bedrooms: min selector (1-5+)
      - Bathrooms: min selector (1-3+)
      - EPC rating: max selector (A-G)
      - New build: toggle
    - All filters sync with URL via useSearch hook (nuqs)
    - Mobile: renders in a Sheet/Drawer triggered by "Filters" button
    - Desktop: sidebar panel (collapsible)
    - "Clear all" button to reset filters
    - Shows active filter count on mobile trigger button

    **`britv3.0/src/components/search/SearchSortBar.tsx`**:
    - Sort dropdown: "Newest first", "Price: low to high", "Price: high to low", "Most relevant"
    - Result count display
    - View toggle: list / map / split icons

    **`britv3.0/src/components/search/SearchResults.tsx`**:
    - Grid of PropertyCard components
    - Responsive: 1 column mobile, 2 columns tablet, 3 columns desktop (in list mode)
    - In split mode: single column
    - Loading skeleton cards while fetching
    - Empty state: "No properties found. Try adjusting your filters."
    - Infinite scroll or "Load more" button for pagination

    **`britv3.0/src/components/search/PropertyCard.tsx`**:
    - Thumbnail image (from thumbnail_url) with fallback placeholder
    - Price (formatted: "250,000" for sale, "1,200 pcm" for rent) with qualifier badge
    - Property type + bedrooms + bathrooms icons
    - Address (line1, city, postcode)
    - Listed date ("2 days ago" via relative time)
    - SaveButton (heart icon, toggles on click)
    - Click navigates to /properties/[slug]
    - Styled with Shadcn Card, responsive, hover state with subtle scale

    **`britv3.0/src/components/properties/SaveButton.tsx`** ("use client"):
    - Heart icon button (filled when saved, outline when not)
    - Uses useSavedProperties hook for optimistic toggle
    - Requires auth -- if not authenticated, redirect to login with return URL
    - Accessible: aria-label "Save property" / "Remove from saved"

    Use Tailwind classes for all styling. Follow Britestate design system (globals.css theme tokens). Use Shadcn components (Card, Button, Input, Select, Sheet, Badge, Skeleton) where available.
