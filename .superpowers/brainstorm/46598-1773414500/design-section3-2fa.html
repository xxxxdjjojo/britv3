Supabase Integration — 2FA
  • Setup: Use supabase.auth.mfa.enroll() → returns TOTP secret + QR SVG. Store factor ID on profiles.totp_factor_id. On verify: supabase.auth.mfa.challengeAndVerify().
  • Code Entry: Post-login MFA challenge via supabase.auth.mfa.challenge() then supabase.auth.mfa.verify(). Supabase handles the AAL2 session upgrade automatically.
  • Role gate: Middleware checks profiles.active_role === 'agent' + profiles.totp_factor_id IS NULL → redirects to 2FA setup page with a dismissible prompt (not forced).
  • Backup codes: Generated client-side (8 × 8-char random codes), hashed with bcrypt, stored in profiles.backup_codes as hashed array. Shown once on setup with download + copy.
  • Lockout: After 5 wrong codes → Supabase rate-limits automatically. UI shows "attempts remaining" count.
