## Critical P0 Security Issues (fix before any new pages)

1. **Admin layout has NO role check** — any authenticated user can browse `/admin/*`
2. **All 6 API routes have no admin role verification** — callable by any authenticated user
3. **`searchUsers()` SQL injection** — query directly interpolated into `.or()` template literal
4. **`suspendUser()` doesn't touch Supabase Auth** — suspended users can still log in
5. **`resolveReport()` ignores the `_adminId` param** — zero audit trail

|---|----------|--------|
| 1 | Admin auth guard placement | Middleware (JWT) for pages + `adminOnly()` per API route |
| 2 | Service file organisation | `src/services/admin/` domain split (9 files) |
| 3 | GDPR fulfilment execution | 202 Accepted + Supabase Edge Function + Realtime status |

| 6 | DB migration files | One file per wave (4 total) |
| 7 | Admin sidebar | Grouped collapsible sections (Radix Collapsible, 6 groups) |
| 8 | Action button feedback | `useAdminAction()` hook — loading state + double-click guard |

| 13 | User search performance | GIN trigram index on `profiles(full_name, email)` |

## New Shared Abstractions

### `src/lib/admin-guard.ts`
```typescript
export async function adminOnly(request: Request) {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return Response.json({ error: 'Unauthorized' }, { status: 401 });

  const { data: profile } = await supabase
    .from('profiles')
    .select('role')
    .eq('id', user.id)
    .single();

  if (profile?.role !== 'admin') {
    return Response.json({ error: 'Forbidden' }, { status: 403 });
  }
  return { user, supabase };
}
```

### `src/lib/audited-admin-action.ts`
```typescript
export async function auditedAdminAction(
  request: Request,
  action: string,
  targetType: string,
  targetId: string,
  fn: (ctx: AdminContext) => Promise<unknown>
): Promise<Response> {
  const ctx = await adminOnly(request);
  if (ctx instanceof Response) return ctx;
  try {
    const result = await fn(ctx);
    return Response.json(result);
  } finally {
    // always logs, even if fn throws
    await logAdminAction(ctx.supabase, {
      adminId: ctx.user.id,
      action, targetType, targetId
    });
  }
}
```

### `src/hooks/useAdminAction.ts`
```typescript
export function useAdminAction() {
  const [isPending, setIsPending] = useState(false);
  const router = useRouter();

  async function execute(url: string, options?: RequestInit) {
    if (isPending) return; // block double-click
    setIsPending(true);
    try {
      const res = await fetch(url, options);
      if (!res.ok) throw new Error('Action failed');
      router.refresh();
    } catch (e) {
      toast.error(String(e)); // use sonner
    } finally {
      setIsPending(false);
    }
  }
  return { execute, isPending };
}
```

## GDPR Fulfilment Flow

```
POST /api/admin/gdpr/[requestId]/fulfil
  1. auditedAdminAction() → verify admin role
  2. Check request.status === 'pending' (idempotency guard)
  3. Mark status = 'in_progress' → return 202 Accepted

Supabase Edge Function: gdpr-fulfil
  4. Query 12 tables with Promise.all:
     profiles, properties, property_images, messages,
     conversations, payments, reviews, content_reports,
     verification_requests, saved_searches, viewing_requests,
     marketplace_enquiries
  5. Assemble JSON export package
  6. Upload to Storage: gdpr-exports/{requestId}/export.json
  7. Create signed URL (7-day expiry)
  8. Send Resend email to user with signed URL
  9. Mark status = 'fulfilled', export_url = signedUrl,
     export_expires_at = now() + 7 days
 10. log admin action: 'gdpr.fulfil'

Error handling:
  - Storage upload fails → status = 'failed', admin notified
  - Resend fails → status = 'email_failed', export ready, retry available
  - User deleted mid-export → skip null tables, flag in metadata
  - Re-triggered on already-fulfilled → no-op (idempotency check)

Admin UI:
  - 'Pending' → click Fulfil → 'Processing...' badge (Realtime sub)
  - → 'Fulfilled' with timestamp
  - → 'Failed' with retry button if error
```

## Error Handling Requirements

### Critical Gaps — All Must Be Fixed

| Codepath | Failure Mode | Required Handler |
|----------|-------------|-----------------|
| `suspendUser()` | Auth update fails after DB update | Rollback DB change (is_suspended back to false) |
| `banUser()` | Same as above | Rollback DB change |
| GDPR Storage upload | Throws | Mark `status='failed'`, log error, alert admin |
| GDPR Resend email | Throws | Mark `status='email_failed'`, export ready, show retry button |
| GDPR user deleted | Null tables | Skip null tables, flag in metadata, still fulfil |
| GDPR duplicate trigger | Already fulfilled | Idempotency check: `if (request.status !== 'pending') return 409` |
| `getFeatureFlags()` cache miss + DB down | Throws | `try/catch` → return `[]`, all flags default off |
| `adminOnly()` DB down | Throws | `try/catch` → return 503 (not 500) |
| PostHog API 429/error | `fetch` throws | Catch → return `null` → page renders `<AnalyticsDegraded />` skeleton |
| Stripe API timeout/error | `fetch` throws | Catch → return `null` → dashboard Suspense shows error state |
| `generateHTML()` unknown node | May throw | Wrap in `try/catch`, return `''` |
