# TrueDeed Visual Audit

Audit date: 2026-06-19.

This audit is based on source scanning, not OCR of screenshots. It identifies visual/user-facing files that still contain legacy brand text.

## User-Facing Surfaces

| Surface | Findings | Files | Risk |
| --- | ---: | ---: | --- |
| Public legal pages | 253 | 20 | High: visible copy, metadata, and legal entity references |
| API/PDF/GDPR exports | 16 | 7 | High: generated user documents and responses |
| Components | 16 | 14 | Medium: shared UI snippets and client flows |
| SEO/json-ld helpers | 4 | 4 | Medium: search metadata and structured data |
| Inngest notifications | 16 | 9 | Medium: background email/notification workflows |

## Visual Review Checklist

- Legal index and each legal detail page show TrueDeed in headings, metadata, JSON-LD, and contact sections.
- Generated quote/invoice PDFs use TrueDeed naming and domains.
- Trust badges, provider panels, auth helpers, install prompts, and comparison UI do not render Britestate.
- SEO titles/descriptions and structured data use `TrueDeed` and `truedeed.co.uk`.
- Notification/email previews generated by background jobs use TrueDeed sender and URLs.

## Current Constraint

The scanner does not inspect pixels in tracked screenshots or binary image files. Visual confirmation still needs a browser pass after source remediation.
